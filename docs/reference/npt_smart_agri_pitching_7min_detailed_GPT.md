# แผนการนำเสนอ NPT Smart Agri Dashboard ช่วง 7 นาทีหลังวิดีโอ Pain Point

> ใช้สำหรับ Pitching Online 18 มิถุนายน 2569  
> โครงการ: **NPT Smart Agri Dashboard**  
> ทีม: **BS-133 NPT Agri NextGen**  
> เป้าหมายของไฟล์นี้: วางโครงนำเสนอหลังจากวิดีโอ 3 นาที ซึ่งวิดีโอได้เล่า Pain Point ไปแล้ว เหลือเวลา 7 นาทีสำหรับพิสูจน์ว่าโครงการนี้ทำได้จริง มีผลลัพธ์จริง และขยายผลได้

## 1. หลักคิดหลักของการนำเสนอ

หลังจากดูเกณฑ์คะแนนแล้ว สิ่งที่ต้องระวังคืออย่าใช้เวลาเล่าปัญหาซ้ำ เพราะปัญหามีเพียง **10 คะแนน** และวิดีโอ 3 นาทีได้ปูเรื่อง Pain Point ไปแล้ว ช่วง 7 นาทีที่เหลือจึงต้องเน้น 3 เรื่องที่ได้คะแนนมากกว่า คือ

1. **ความสร้างสรรค์และความเป็นไปได้จริง** 35 คะแนน
2. **ผลสัมฤทธิ์และการขยายผล** 30 คะแนน
3. **ทีมเวิร์กและการบูรณาการ** 25 คะแนน

ดังนั้นแกนการเล่าใน 7 นาทีควรเป็น

> จากปัญหาข้อมูลกระจัดกระจาย เราพัฒนา NPT Smart Agri Dashboard ให้เป็นศูนย์ข้อมูลเกษตรกลางของจังหวัดนครปฐม ที่รวม Dashboard, GIS Map, Data Management, Global Search และ AI Chatbot ไว้ในระบบเดียว ใช้งานได้จริง แบ่งสิทธิ์ข้อมูลได้จริง ลดภาระเจ้าหน้าที่ เพิ่มความเร็วในการตัดสินใจของผู้บริหาร และสามารถขยายเป็นต้นแบบให้จังหวัดอื่นได้

ประโยคนี้คือแกนของการนำเสนอทั้งหมด ไม่ต้องพูดทุกคำตามนี้ก็ได้ แต่ทุกส่วนที่พูดควรพากรรมการกลับมาที่แกนนี้เสมอ

## 3. สิ่งที่ควรสื่อสารให้กรรมการจำได้

กรรมการควรจำโครงการนี้ได้ด้วย 5 คำหลัก

1. **ฐานข้อมูลกลาง**  
   รวมข้อมูลการเกษตรจังหวัดนครปฐมที่เคยกระจัดกระจายให้มาอยู่ในจุดเดียว

2. **Dashboard และแผนที่**  
   เปลี่ยนข้อมูลตารางให้กลายเป็นภาพรวม กราฟ และ GIS Map ที่ผู้บริหารและเจ้าหน้าที่เข้าใจเร็ว

3. **AI ช่วยงานเจ้าหน้าที่**  
   ใช้ AI Chatbot ช่วยค้นหา สรุป และตอบคำถามจากข้อมูลในระบบ ลดการไล่เปิดไฟล์หลายชุด

4. **ปลอดภัยและแยกสิทธิ์**  
   ข้อมูลสาธารณะเปิดเผยได้ ข้อมูลภายในต้องเข้าสู่ระบบ ข้อมูลจำกัดเข้าถึงเฉพาะผู้เกี่ยวข้อง

5. **ขยายผลเป็นต้นแบบ**  
   โครงสร้างนี้สามารถปรับใช้กับจังหวัดอื่นได้ โดยเปลี่ยนชุดข้อมูลและบริบทพื้นที่

---

## 4. โครงเรื่องหลัก 7 นาที

### ภาพรวมการเล่า

1. เปิดหลังวิดีโอ: ปัญหาเล่าไปแล้ว ระบบนี้คือคำตอบ
2. อธิบายระบบ: ระบบเดียวรองรับผู้บริหาร เจ้าหน้าที่ ประชาชน
3. พิสูจน์ว่าใช้จริง: Dashboard, Map, Search, AI, Data Management
4. อธิบายข้อมูล: ข้อมูลภายใน + ข้อมูลภายนอก + ฐานข้อมูลกลาง
5. แสดงผลลัพธ์: ลดเวลา ตัดสินใจเร็ว บริการเร็ว จัดการความรู้ได้
6. บูรณาการทีม: แต่ละกลุ่มงานเป็นเจ้าของข้อมูลในมาตรฐานเดียวกัน
7. ปิด: จากระบบจังหวัด สู่ Nakhon Pathom Model

## 8. หน้าใหม่ที่ควรทำเพิ่ม

สไลด์เดิมมีภาพเยอะและเนื้อหาครบ แต่ยังมีบางหน้าที่ควรทำใหม่เพื่อให้ตรงเกณฑ์มากขึ้น

### หน้าใหม่ที่ 1: ระบบนี้คืออะไร ใครใช้

**หัวข้อ:**  
NPT Smart Agri Dashboard ทำงานให้ใครบ้าง

**โครงภาพ:**

กลาง: NPT Smart Agri Dashboard  
เชื่อมออก 3 กลุ่ม

1. ผู้บริหาร
   - ดูภาพรวมจังหวัด
   - ติดตามสถานการณ์
   - ตัดสินใจจากข้อมูล

2. เจ้าหน้าที่
   - ค้นหาและจัดการข้อมูล
   - ลดการรวบรวมไฟล์ซ้ำ
   - ใช้ AI ช่วยสรุป

3. ประชาชน/เกษตรกร
   - เข้าถึงข้อมูลสาธารณะ
   - ดูแหล่งเรียนรู้ แปลงใหญ่ วิสาหกิจชุมชน
   - ใช้ข้อมูลวางแผนเบื้องต้น

**เหตุผลที่ควรมี:**  
ช่วยให้กรรมการเข้าใจระบบก่อนลงรายละเอียดฟีเจอร์

---

### หน้าใหม่ที่ 2: ฟีเจอร์หลัก 5 อย่าง

**หัวข้อ:**  
5 เครื่องมือหลักที่เปลี่ยนข้อมูลให้ใช้ประโยชน์ได้

**เนื้อหา:**

1. Dashboard  
   สรุปภาพรวมข้อมูลสำคัญของจังหวัด

2. Smart Map  
   แสดงข้อมูลเชิงพื้นที่และจุดสำคัญทางการเกษตร

3. Data Management  
   บันทึก แก้ไข ค้นหา และนำเข้าข้อมูลของแต่ละกลุ่มงาน

4. Global Search  
   ค้นหาข้ามตาราง ลดเวลาการไล่เปิดไฟล์หลายชุด

5. AI Chatbot  
   ช่วยถามตอบและสรุปข้อมูลจากฐานข้อมูล

**เหตุผลที่ควรมี:**  
ช่วยตอบเกณฑ์สร้างสรรค์/ทำได้จริงแบบชัดที่สุด

---

### หน้าใหม่ที่ 3: 3 Use Cases

**หัวข้อ:**  
3 กรณีใช้งานที่พิสูจน์ผลลัพธ์

**เนื้อหา:**

1. เชื่อมโยงตลาดและผลผลิต  
   ใช้ข้อมูลแปลงใหญ่ พื้นที่การผลิต และกลุ่มเกษตรกร ช่วยจับคู่ความต้องการตลาด

2. สนับสนุนการตัดสินใจผู้บริหาร  
   Dashboard และ GIS ช่วยเห็นพื้นที่เป้าหมาย ความเสี่ยง และสถานการณ์เร่งด่วน

3. จัดการความรู้ในสำนักงาน  
   ลดการพึ่งพาไฟล์ส่วนตัวและความรู้ที่อยู่กับคนใดคนหนึ่ง

**เหตุผลที่ควรมี:**  
ตอบเกณฑ์ผลสัมฤทธิ์และการขยายผลได้ตรงมาก

---

### หน้าใหม่ที่ 4: บูรณาการทั้งสำนักงาน

**หัวข้อ:**  
ข้อมูลของทุกกลุ่มงาน เชื่อมเป็นภาพเดียว

**โครงภาพ:**

กลาง: ฐานข้อมูลกลางจังหวัดนครปฐม

เชื่อมเข้า 5 กลุ่มงาน

1. ฝ่ายบริหารทั่วไป  
   บุคลากร พัสดุ งบประมาณ

2. กลุ่มยุทธศาสตร์และสารสนเทศ  
   ทะเบียนเกษตรกร พื้นที่เกษตร GIS ศูนย์เรียนรู้

3. กลุ่มส่งเสริมและพัฒนาการผลิต  
   แปลงใหญ่ GAP ผลผลิตพืช

4. กลุ่มส่งเสริมและพัฒนาเกษตรกร  
   Smart Farmer วิสาหกิจชุมชน กลุ่มเกษตรกร ท่องเที่ยวเกษตร

5. กลุ่มอารักขาพืช  
   จุดความร้อน แปลงพยากรณ์ ศจช. ศดปช. ศูนย์ดินปุ๋ย

**เหตุผลที่ควรมี:**  
ตอบเกณฑ์ทีมเวิร์ก/บูรณาการ 25 คะแนนได้ชัด

## 9. สิ่งที่ควรตัดออกจากการพูดหลัก

- React version
- Vite version
- routing รายหน้า
- package manager
- โครงสร้าง component
- รายละเอียดการ build/deploy

**ควรพูดแทน:**

- เทคโนโลยีเลือกมาเพื่อให้ใช้งานจริงได้เร็ว

- รองรับการขยายผลและดูแลต่อได้

ห้ามวนกลับไปเล่าปัญหาเกิน 30 วินาที เพราะวิดีโอทำหน้าที่นี้แล้ว

## 10. วิธีเล่าให้ได้คะแนนตามเกณฑ์

### เกณฑ์ 1: ปัญหาชัดเจน 10 คะแนน

**สิ่งที่ต้องพูด:**

- ข้อมูลกระจัดกระจายหลายไฟล์
- งานซ้ำซ้อนในการรวบรวมข้อมูล
- ตัดสินใจช้าเพราะข้อมูลไม่อยู่ในจุดเดียว
- มีความเสี่ยงเรื่องข้อมูลส่วนบุคคลและการเผยแพร่ข้อมูล

**เวลา:** 20-30 วินาที

**ประโยคตัวอย่าง:**

“ปัญหาหลักคือข้อมูลเกษตรจังหวัดไม่ได้ขาด แต่กระจัดกระจาย อยู่หลายไฟล์ หลายคน หลายกลุ่มงาน ทำให้การเรียกใช้ข้อมูลเพื่อบริหารงานต้องใช้เวลาและเกิดความซ้ำซ้อน”

---

### เกณฑ์ 2: สร้างสรรค์/ทำได้จริง 35 คะแนน

**สิ่งที่ต้องพูด:**

- เป็น Web Application ที่ทำงานได้จริง
- มี Public Portal และ Internal Dashboard
- มี Dashboard, GIS Map, Search, AI Chatbot, Data Management
- เชื่อมฐานข้อมูลกลางและข้อมูลภายนอก
- แยกระดับข้อมูลและสิทธิ์ผู้ใช้
- ใช้เทคโนโลยีเหมาะสมกับทรัพยากร เช่น cloud database, web app, serverless proxy

**เวลาที่ควรใช้:** 2.5-3 นาที

**ประโยคตัวอย่าง:**

“ความสร้างสรรค์ของระบบนี้อยู่ที่การรวมเครื่องมือที่เจ้าหน้าที่ต้องใช้จริงไว้ในระบบเดียว ทั้ง Dashboard, แผนที่, ตารางจัดการข้อมูล, ระบบค้นหาข้ามตาราง และ AI Chatbot โดยทั้งหมดเชื่อมกับฐานข้อมูลกลางเดียวกัน ทำให้ข้อมูลหนึ่งชุดถูกใช้ซ้ำได้หลายงาน ไม่ต้องทำไฟล์ใหม่แยกตามรายงานทุกครั้ง”

---

### เกณฑ์ 3: ผลสัมฤทธิ์/ขยายผลได้ 30 คะแนน

**สิ่งที่ต้องพูด:**

- ลดเวลาการรวบรวมข้อมูล
- ลดการเปิดไฟล์หลายชุด
- ลดการถามข้อมูลซ้ำระหว่างกลุ่มงาน
- ผู้บริหารตัดสินใจจากข้อมูลเร็วขึ้น
- เจ้าหน้าที่ใหม่เรียนรู้งานเร็วขึ้น
- ประชาชนเข้าถึงข้อมูลสาธารณะได้
- ขยายผลไปจังหวัดอื่นได้เพราะโครงสร้างแยกข้อมูลเป็นโมดูล

**เวลาที่ควรใช้:** 2 นาที

**ประโยคตัวอย่าง:**

“ผลลัพธ์ที่คาดหวังคือเปลี่ยนเวลาที่เคยใช้กับการรวบรวมไฟล์ ให้กลายเป็นเวลาที่ใช้วิเคราะห์และตัดสินใจ ผู้บริหารเห็นภาพรวมเร็วขึ้น เจ้าหน้าที่ตอบคำถามได้เร็วขึ้น และข้อมูลบางส่วนสามารถเปิดเป็น Open Data ให้ประชาชนใช้ประโยชน์ได้ โดยยังคงแยกข้อมูลภายในและข้อมูลส่วนบุคคลออกจากกัน”

---

### เกณฑ์ 4: ทีมเวิร์ก/บูรณาการ 25 คะแนน

**สิ่งที่ต้องพูด:**

- แต่ละกลุ่มงานเป็นเจ้าของข้อมูลของตัวเอง
- ระบบรวมข้อมูลทุกกลุ่มงานเข้าสู่มาตรฐานเดียว
- มีการแบ่งสิทธิ์ตามบทบาทและกลุ่มงาน
- ทำให้สำนักงานเห็นข้อมูลชุดเดียวกัน
- ข้อมูลภายนอก เช่น อากาศ จุดความร้อน ราคา ช่วยเติมมุมมองให้ข้อมูลภายใน

**เวลาที่ควรใช้:** 1.5 นาที

**ประโยคตัวอย่าง:**

“ระบบนี้ตั้งใจให้ข้อมูลไม่ได้อยู่ที่คนใดคนหนึ่ง แต่ให้แต่ละกลุ่มงานเป็นเจ้าของข้อมูลของตัวเอง และเชื่อมเข้าฐานข้อมูลกลางร่วมกัน เมื่อทุกฝ่ายใช้มาตรฐานเดียวกัน ผู้บริหารและเจ้าหน้าที่จะเห็นภาพเดียวกัน ลดการส่งไฟล์ซ้ำ ลดการถามข้อมูลซ้ำ และสร้างวัฒนธรรมการใช้ข้อมูลร่วมกันในสำนักงาน”

## 13. ตัวชี้วัดที่ควรพูดหรือใส่ท้ายสไลด์

### ตัวชี้วัดระยะเริ่มต้น

1. จำนวนชุดข้อมูลที่นำเข้าระบบ
2. จำนวนกลุ่มงานที่มีข้อมูลในระบบ
3. จำนวนหน้า dashboard หรือ module ที่พร้อมใช้งาน
4. จำนวนผู้ใช้ภายในที่ทดลองใช้งาน
5. จำนวนข้อมูลสาธารณะที่เปิดให้ประชาชนดูได้
6. จำนวน API หรือข้อมูลภายนอกที่เชื่อมต่อแล้ว
7. เวลาที่ใช้ในการค้นหาและสรุปข้อมูลเทียบกับรูปแบบเดิม

### ตัวชี้วัดระยะกลาง

1. อัตราการอัปเดตข้อมูลตามรอบที่กำหนด
2. จำนวนการค้นหาข้อมูลผ่านระบบ
3. จำนวนคำถามผ่าน AI Chatbot
4. จำนวนรายงานหรือการตัดสินใจที่ใช้ข้อมูลจากระบบ
5. ความพึงพอใจของผู้บริหารและเจ้าหน้าที่
6. จำนวนหน่วยงานหรืออำเภอที่นำข้อมูลไปใช้ต่อ

### ตัวชี้วัดระยะขยายผล

1. จำนวนจังหวัดหรือหน่วยงานที่นำโมเดลไปปรับใช้
2. จำนวนชุดข้อมูลมาตรฐานที่ใช้ร่วมกันได้
3. จำนวนบริการข้อมูลสาธารณะที่ประชาชนเข้าถึงได้
4. ระยะเวลาการจัดทำรายงานระดับจังหวัดที่ลดลง
5. ความต่อเนื่องของการอัปเดตข้อมูลหลังจบโครงการ

  ควรพูดว่า “ลดเวลาการตามไฟล์และรวบรวมข้อมูลซ้ำ”

- ใช้แทนเจ้าหน้าที่

## 16. สคริปต์ฉบับกระชับ 7 นาที

> ใช้กรณีต้องพูดแบบต่อเนื่อง ไม่เปิด demo สด

“จากวิดีโอเมื่อสักครู่ เราจะเห็นปัญหาหลักของงานข้อมูลการเกษตรระดับจังหวัด คือข้อมูลไม่ได้ขาด แต่กระจัดกระจาย อยู่หลายไฟล์ หลายกลุ่มงาน และเมื่อต้องใช้ข้อมูลประกอบการตัดสินใจ เจ้าหน้าที่ต้องใช้เวลารวบรวม ตรวจสอบ และสรุปใหม่ซ้ำ ๆ ทีม BS-133 NPT Agri NextGen จึงพัฒนา NPT Smart Agri Dashboard ให้เป็นศูนย์ข้อมูลการเกษตรอัจฉริยะของจังหวัดนครปฐม

ระบบนี้ออกแบบเป็น Web Application ที่รองรับผู้ใช้ 3 กลุ่มหลัก ได้แก่ ประชาชนและเกษตรกร เจ้าหน้าที่ และผู้บริหาร ประชาชนสามารถเข้าถึงข้อมูลสาธารณะ เช่น แปลงใหญ่ วิสาหกิจชุมชน Smart Farmer แหล่งท่องเที่ยวเชิงเกษตร และข้อมูลสถานการณ์บางส่วน เจ้าหน้าที่ใช้ค้นหา บันทึก แก้ไข ตรวจสอบ และจัดการข้อมูลของแต่ละกลุ่มงาน ส่วนผู้บริหารใช้ดูภาพรวมจังหวัด แผนที่ สถานการณ์ และข้อมูลประกอบการตัดสินใจแบบรวดเร็ว

ฟีเจอร์หลักของระบบมี 5 ส่วน ส่วนแรกคือ Dashboard ที่สรุปข้อมูลสำคัญเป็นกราฟ ตัวเลข และตารางที่เข้าใจง่าย ส่วนที่สองคือ Smart Map ที่แสดงข้อมูลเชิงพื้นที่ เช่น แปลงใหญ่ ศูนย์เรียนรู้ แหล่งท่องเที่ยวเชิงเกษตร และจุดความร้อน ส่วนที่สามคือ Data Management สำหรับให้เจ้าหน้าที่จัดการข้อมูลของแต่ละกลุ่มงาน ส่วนที่สี่คือ Global Search สำหรับค้นหาข้ามตาราง ลดเวลาการเปิดไฟล์หลายชุด และส่วนที่ห้าคือ AI Chatbot ที่ช่วยถามตอบและสรุปข้อมูลจากฐานข้อมูลในระบบ

เบื้องหลังของระบบนี้คือแนวคิดฐานข้อมูลกลาง ข้อมูลที่เคยอยู่ใน Excel, Google Sheet, รายงาน PDF หรืออยู่แยกตามกลุ่มงาน จะถูกจัดระเบียบ ทำความสะอาด และนำเข้าเป็นโครงสร้างเดียวกัน จากนั้นข้อมูลหนึ่งชุดสามารถนำไปใช้ต่อได้หลายทาง ทั้ง Dashboard, Map, Search และ AI Chatbot ขณะเดียวกัน ข้อมูลภายนอก เช่น สภาพอากาศ คุณภาพอากาศ จุดความร้อน หรือราคาสินค้าเกษตร สามารถเชื่อมผ่าน API หรือ proxy ที่ปลอดภัย เพื่อให้ข้อมูลมีความทันสมัยและใช้ประกอบการตัดสินใจได้ดีขึ้น

ในด้านความพร้อม ปัจจุบันระบบไม่ได้อยู่ในระดับแนวคิดเท่านั้น แต่มีความพร้อมสำหรับการทดลองใช้งานจริงแล้ว โดยเริ่มนำเข้าข้อมูลหลักมากกว่า 13 ชุดข้อมูล เชื่อมต่อข้อมูลสำคัญ เช่น สภาพอากาศและข้อมูลจุดความร้อน และมีโครงสร้างรองรับการใช้งานของทั้งผู้บริหาร เจ้าหน้าที่ และประชาชนในส่วนที่เป็นข้อมูลสาธารณะ

ผลลัพธ์ที่คาดหวังเห็นได้จาก 3 กรณีใช้งาน กรณีแรกคือด้านตลาดและผลผลิต หากมีโรงงานหรือผู้ประกอบการต้องการรับซื้อผลผลิตในพื้นที่ ระบบสามารถใช้ข้อมูลแปลงใหญ่ พื้นที่การผลิต และข้อมูลกลุ่มเกษตรกร เพื่อช่วยคัดกรองกลุ่มเป้าหมายได้เร็วขึ้น กรณีที่สองคือการตัดสินใจของผู้บริหาร เช่น การวางแผนงบประมาณ การติดตามภัยแล้ง PM2.5 จุดความร้อน หรือสถานการณ์ด้านการผลิต ผู้บริหารสามารถดูข้อมูลผ่าน Dashboard และ Map ได้ทันที เห็นพื้นที่เป้าหมายและความเสี่ยงโดยไม่ต้องรอรวบรวมเอกสารใหม่ทุกครั้ง กรณีที่สามคือการจัดการความรู้ของสำนักงาน เมื่อเจ้าหน้าที่เกษียณ ย้ายงาน หรือมีเจ้าหน้าที่ใหม่เข้ามา ระบบช่วยให้ข้อมูลและประวัติการทำงานสำคัญถูกรวบรวมไว้ในที่เดียว ลดการพึ่งพาไฟล์ส่วนตัวหรือความรู้ที่อยู่กับคนใดคนหนึ่ง

อีกประเด็นสำคัญคือระบบนี้ออกแบบให้เกิดการบูรณาการทั้งสำนักงาน แต่ละกลุ่มงานเป็นเจ้าของข้อมูลของตัวเอง กลุ่มยุทธศาสตร์และสารสนเทศดูข้อมูลภาพรวม พื้นที่ และทะเบียนเกษตรกร กลุ่มส่งเสริมและพัฒนาการผลิตดูข้อมูลแปลงใหญ่ GAP และผลผลิตพืช กลุ่มส่งเสริมและพัฒนาเกษตรกรดูข้อมูล Smart Farmer วิสาหกิจชุมชน และกลุ่มเกษตรกร กลุ่มอารักขาพืชดูข้อมูลจุดความร้อน แปลงพยากรณ์ ศจช. และ ศดปช. ส่วนฝ่ายบริหารทั่วไปดูข้อมูลบุคลากร พัสดุ งบประมาณ และการบริหารสำนักงาน เมื่อทุกกลุ่มงานใช้มาตรฐานข้อมูลร่วมกัน ผู้บริหารและเจ้าหน้าที่จะเห็นภาพเดียวกัน ลดการส่งไฟล์ซ้ำ ลดการถามข้อมูลซ้ำ และสร้างวัฒนธรรมการใช้ข้อมูลร่วมกันในสำนักงาน

เป้าหมายของ NPT Smart Agri Dashboard คือการยกระดับข้อมูลการเกษตรจังหวัดนครปฐมให้เป็นฐานข้อมูลกลางที่ใช้ตัดสินใจได้จริง ลดภาระงานเจ้าหน้าที่ และเปิดทางให้ประชาชนเข้าถึงข้อมูลที่เป็นประโยชน์ ในระยะต่อไป โครงสร้างนี้สามารถพัฒนาเป็น Nakhon Pathom Model เพื่อให้จังหวัดอื่นนำแนวทางไปปรับใช้ได้ตามบริบทของตนเอง ทั้งด้านการบริหารงาน การบริการเกษตรกร และการตัดสินใจเชิงนโยบายครับ”

## 19. คำแนะนำสุดท้ายสำหรับการนำเสนอ

### ควรทำ

1. เปิดให้เร็ว  
   อย่าใช้เวลานานกับชื่อโครงการและที่มา

2. พูดด้วยภาษางานจริง  
   ใช้คำว่า “ลดการตามไฟล์”, “ลดการถามข้อมูลซ้ำ”, “ผู้บริหารเห็นภาพรวมเร็วขึ้น” จะจับต้องได้กว่า “ยกระดับ digital transformation” อย่างเดียว

3. ย้ำว่าเริ่มจากข้อมูลที่มีอยู่แล้ว  
   ทำให้กรรมการเห็นว่าโครงการทำได้จริง ไม่ใช่ต้องรอระบบใหญ่

4. เล่า use case เป็นเรื่อง  
   เช่น ตลาดต้องการผลผลิต ผู้บริหารต้องการตัดสินใจ เจ้าหน้าที่ใหม่ต้องเรียนรู้งาน

5. ปิดด้วยโมเดลขยายผล  
   ให้กรรมการรู้ว่าโครงการมีอนาคต ไม่จบที่ dashboard จังหวัดเดียว

### ไม่ควรทำ

1. อย่าเล่าปัญหาซ้ำเกิน 30 วินาที
2. อย่าโชว์ทุกหน้า ทุกเมนู ทุกฟีเจอร์
3. อย่าพูดศัพท์เทคนิคเยอะเกินไป
4. อย่าบอกว่า AI ทำได้ทุกอย่าง
5. อย่าใช้ตัวเลขผลลัพธ์ที่ไม่มีหลักฐานรองรับ
6. อย่าปิดด้วยคำกว้าง ๆ ให้ปิดด้วยภาพขยายผลที่ชัด

## 28. สูตรจำง่ายก่อนขึ้นเวที

ให้จำลำดับนี้

```text
Pain Point → Platform → Proof → Impact → Team → Scale
```

แปลเป็นภาษาเล่าจริง

1. Pain Point  
   ปัญหาเล่าไปแล้วในวิดีโอ

2. Platform  
   ระบบนี้คือฐานข้อมูลกลางและ web application

3. Proof  
   มี dashboard, map, AI, data management และข้อมูลจริง

4. Impact  
   ลดงานซ้ำ ตัดสินใจเร็ว บริการดีขึ้น

5. Team  
   ทุกกลุ่มงานเป็นเจ้าของข้อมูลร่วมกัน

6. Scale  
   ขยายเป็น Nakhon Pathom Model ได้

## 29. สุดท้าย สิ่งที่ต้องทำให้กรรมการเชื่อ

กรรมการต้องเชื่อ 4 เรื่องนี้

1. **ปัญหาจริง**  
   งานข้อมูลเกษตรจังหวัดมี pain point จริง และวิดีโอแสดงให้เห็นแล้ว

2. **ระบบจริง**  
   มี web app, dashboard, map, AI, database และข้อมูลจริง ไม่ใช่แผนในกระดาษ

3. **ผลลัพธ์จริง**  
   ลดเวลาทำงาน ลดการตามไฟล์ ช่วยตัดสินใจ และช่วยถ่ายทอดความรู้ในสำนักงาน

4. **ขยายผลจริง**  
   โครงสร้างระบบแยกเป็นโมดูล ใช้กับจังหวัดอื่นได้เมื่อปรับข้อมูลและบริบทพื้นที่

ถ้าการนำเสนอทำให้กรรมการเชื่อ 4 เรื่องนี้ได้ คะแนนจะไปอยู่ในส่วน 35 + 30 + 25 เป็นหลัก ซึ่งเป็นคะแนนส่วนใหญ่ของเกณฑ์ทั้งหมด
