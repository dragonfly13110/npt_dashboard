## 1. บทนำและวัตถุประสงค์

### 1.1 ที่มาของโครงการ

NPT Smart Agri Dashboard เกิดจากปัญหาจริงในการทำงานของสำนักงานเกษตรจังหวัดนครปฐม ที่พบว่ามีข้อมูลเกษตรกระจัดกระจายอยู่ในหลายระบบ หลายกลุ่มงาน ทำให้:

- เจ้าหน้าที่ใช้เวลาในการรวบรวมข้อมูลทำรายงานนาน 3-5 วัน ต่อ 1 ชุดรายงาน
- ผู้บริหารไม่สามารถเข้าถึงข้อมูล Real-time เพื่อตัดสินใจได้ทันที
- การขอข้อมูลจากอำเภอหรือกลุ่มงานต่าง ๆ ใช้เวลานาน ติดตามยาก
- ข้อมูลซ้ำซ้อน ไม่เป็นปัจจุบัน และตรวจสอบความถูกต้องได้ยาก

### 1.2 วัตถุประสงค์การพัฒนา

1. **รวมศูนย์ข้อมูล** จาก 5 กลุ่มงาน + 7 อำเภอ ไว้ในระบบเดียว
2. **ลดเวลาการทำงาน** ของเจ้าหน้าที่ จากวันเหลือนาที
3. **สนับสนุนการตัดสินใจ** ของผู้บริหารด้วยข้อมูล Real-time
4. **ยกระดับบริการ** เกษตรกรด้วยการเข้าถึงข้อมูลที่รวดเร็ว
5. **สร้างต้นแบบดิจิทัล** ที่สามารถขยายผลไปยังจังหวัดอื่นได้

### 1.3 กลุ่มเป้าหมาย

- **ผู้ใช้หลัก:** ผู้บริหารสำนักงานเกษตรจังหวัด, หัวหน้ากลุ่มงาน, เกษตรอำเภอ
- **ผู้ใช้รอง:** เจ้าหน้าที่วิเคราะห์ข้อมูล, เจ้าหน้าที่ส่งเสริมการเกษตร
- **ผู้ใช้ทั่วไป:** เกษตรกร, วิสาหกิจชุมชน, สถาบันเกษตรกร, ประชาชน

### 1.4 ขอบเขตการนำเสนอ (7 นาที)

เนื่องจากมีเวลาจำกัดเพียง 7 นาที (หลังจากวิดีโอ 3 นาทีแรกเกี่ยวกับ Pain Point) การนำเสนอครั้งนี้จะโฟกัสที่:

- ✅ **Solution Demo** - แสดงระบบจริง 4 ฟีเจอร์หลัก
- ✅ **Technology & Innovation** - เทคโนโลยีและนวัตกรรมที่ใช้
- ✅ **Readiness & Data Integration** - ความพร้อมและข้อมูลที่รวมแล้ว
- ✅ **Value & Impact** - ผลลัพธ์ที่วัดได้เป็นตัวเลข
- ✅ **Expansion & Sustainability** - แผนขยายผลและความยั่งยืน

❌ **สิ่งที่ตัดออก:**

- รายละเอียด Code หรือ Database Schema
- ปัญหา Technical Debt ที่กำลังแก้ไข
- ฟีเจอร์รองที่ยังไม่สมบูรณ์
- ประวัติการพัฒนาแบบละเอียด

- คะแนนความเสี่ยง (คำนวณจากหลายปัจจัย)

#### Recommended Actions (กลางล่าง)

- Action ที่แนะนำ
- เหตุผล
- Dataset ที่ใช้อ้างอิง
- ปุ่ม "ดำเนินการ" (ถ้ามี)

#### Quick Charts (ขวา)

กราฟเล็กๆ 3-4 กราฟ:

- แนวโน้มราคาพืชเศรษฐกิจ
- จำนวน Hotspot รายสัปดาห์
- สถานะน้ำในอ่างเก็บน้ำ
- จำนวนกิจกรรมเกษตรกร

**การ Demo:**

1. เปิดหน้า Situation Room

- **พื้นที่:** นครปฐม (Lat: 13.8199, Lon: 100.0428)

#### 2. Open-Meteo Air Quality API

- **URL:** `https://air-quality-api.open-meteo.com/v1/air-quality`
- **ข้อมูล:** PM2.5, PM10, AQI, O3, NO2
- **ความถี่:** ทุก 1 ชั่วโมง
- **การใช้:** แสดงคุณภาพอากาศและเตือนภัย

#### 3. MOC Prices API (กรมการค้าภายใน)

- **URL:** `https://price.moc.go.th/api/...`
- **ข้อมูล:** ราคาสินค้าเกษตร (ข้าว, มัน, อ้อย, ข้าวโพด)
- **ความถี่:** ทุกวันทำการ
- **การใช้:** แสดงราคาปัจจุบันและแนวโน้ม

#### 4. GISTDA VIIRS API (จุดความร้อน)

- **URL:** `https://fire.gistda.or.th/api/...`
- **ข้อมูล:** พิกัด Hotspot, ความรุนแรง, เวลา
- **ความถี่:** ทุกวัน (ดาวเทียมผ่าน)
- **การใช้:** เฝ้าระวังไฟป่าและการเผา

#### 5. RID Reservoir API (กรมชลประทาน)

- **URL:** `https://app.rid.go.th/dam/api/...`

- **ข้อมูล:** ข่าวจากกรมส่งเสริมการเกษตรและหน่วยงาน
- **ความถี่:** ทุก 30 นาที
- **การใช้:** แสดงข่าวล่าสุด

#### 7. KKU Chatbot API (มหาวิทยาลัยขอนแก่น)

- **URL:** `https://chatbot.kku.ac.th/api/...`
- **ข้อมูล:** คำตอบคำถามเกษตรสาธารณะ

### 12.2 หมวดเทคโนโลยีและนวัตกรรม (15 ข้อ)

**Q11: ทำไมถึงเลือก React และ Vite?**

**Q17: ระบบรองรับ Offline ไหม?**

## 15. กลยุทธ์ชนะใจกรรมการ

### 15.1 เน้น Impact มากกว่า Feature

**ผิด:** "ระบบมี 25 Dataset, 7 API, AI Chatbot..."
**ถูก:** "ลดเวลาทำรายงานจาก 3-5 วัน เหลือ 5-10 นาที หรือ 99%"

**เทคนิค:**

- เริ่มด้วย Impact
- ตามด้วย Feature ที่สนับสนุน
- จบด้วย Impact อีกครั้ง

### 15.2 Show, Don't Just Tell

**ผิด:** พูดอย่างเดียว
**ถูก:** Demo ระบบจริง + พูดประกอบ

**เทคนิค:**

- ให้ระบบพูดแทนเรา
- ชี้ให้เห็นตัวเลข
- ให้กรรมการเห็นภาพ

### 15.3 เล่าเป็นเรื่อง (Storytelling)

**โครงสร้างเรื่อง:**

1. **Hero:** เจ้าหน้าที่เกษตร/เกษตรกร
2. **Problem:** ข้อมูลกระจัดกระจาย เสียเวลา
3. **Guide:** ทีมพัฒนา
4. **Solution:** NPT Smart Agri Dashboard
5. **Result:** ลดเวลา 99%, ตัดสินใจเร็วขึ้น
6. **Call to Action:** ขยายผลจังหวัดอื่น

**เทคนิค:**

- ใช้ภาษาเรียบง่าย
- มีอารมณ์ร่วม
- ทำให้เห็นภาพ

### 15.4 ย้ำความพร้อมใช้งาน

**ข้อความที่ต้องย้ำ:**

- "ใช้งานจริงแล้ว"
- "ไม่ใช่ Prototype"
- "มีผู้ใช้งาน X คน"
- "ข้อมูล X ชุด"

**หลักฐาน:**

- Show ระบบจริง
- Show ข้อมูลจริง
- Show User จริง (ถ้ามี)

### 15.5 ตอบโจทย์ทุกเกณฑ์

| เกณฑ์                | วิธีตอบโจทย์              |
| -------------------- | ------------------------- |
| ปัญหาชัดเจน          | ทวน Pain Point, ใช้ตัวเลข |
| สร้างสรรค์/ทำได้จริง | Demo, เทคโนโลยี, AI       |
| ผลสัมฤทธิ์/ขยายผลได้ | Before/After, Roadmap     |
| ทีมเวิร์ก/บูรณาการ   | แนะนำทีม, หน่วยงานร่วม    |

### 15.6 สร้างความประทับใจ

**เปิดให้จำ:**

- เริ่มต้นด้วยตัวเลข Impact
- หรือ Story สั้นๆ

**ปิดให้จำ:**

- สรุป Key Messages 3 ข้อ
- หรือ Quote กินใจ

**ระหว่างนำเสนอ:**

- สบตากรรมการ
- ยิ้มแย้ม
- มั่นใจแต่ไม่โอ้อวด
