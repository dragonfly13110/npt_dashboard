# เริ่มต้นสำหรับมือใหม่: ทำเว็บ NPT Dashboard ตั้งแต่ศูนย์จนออนไลน์

คู่มือนี้เขียนสำหรับเจ้าหน้าที่หรือนักวิชาการเกษตรที่ไม่ถนัดงานคอมพิวเตอร์มาก่อน แต่อยากติดตั้ง ทดลอง และเผยแพร่เว็บ `NPT Smart Agri Dashboard` ได้ด้วยตนเอง ตั้งแต่เตรียมเครื่อง จนเปิดเว็บออนไลน์ได้

เป้าหมายของบทนี้ไม่ใช่การอธิบายโค้ดเชิงลึก แต่เป็นการพาทำงานจริงทีละขั้น อ่านแล้วทำตามได้ แม้ไม่เคยทำเว็บมาก่อน

## ภาพรวมแบบสั้น

ระบบนี้ประกอบด้วย 3 ส่วนหลัก

| ส่วน     | ความหมายแบบง่าย        | ใช้ทำอะไร                                            |
| -------- | ---------------------- | ---------------------------------------------------- |
| โค้ดเว็บ | ไฟล์โปรแกรมของหน้าเว็บ | แสดง Dashboard, ตาราง, แผนที่, Search และ AI Chatbot |
| Supabase | ฐานข้อมูลออนไลน์       | เก็บข้อมูลเกษตร ผู้ใช้ สิทธิ์ และตารางต่าง ๆ         |
| Netlify  | ที่ฝากเว็บออนไลน์      | ทำให้คนอื่นเปิดเว็บผ่านลิงก์ได้                      |

ลำดับงานทั้งหมดคือ

```text
เตรียมเครื่อง
-> ดาวน์โหลดโค้ด
-> เปิดเว็บในเครื่องตัวเอง
-> สร้างฐานข้อมูล Supabase
-> ใส่ค่าการเชื่อมต่อ
-> ทดสอบระบบ
-> ส่งเว็บขึ้น Netlify
-> ตรวจหลังออนไลน์
```

## คำศัพท์ที่ต้องรู้ก่อน

| คำ                    | แปลแบบง่าย                                         |
| --------------------- | -------------------------------------------------- |
| Terminal / PowerShell | หน้าต่างสำหรับพิมพ์คำสั่ง                          |
| GitHub                | ที่เก็บโค้ดของโครงการ                              |
| Clone                 | ดาวน์โหลดโค้ดจาก GitHub ลงเครื่อง                  |
| Node.js               | โปรแกรมพื้นฐานที่ใช้รันเว็บ React                  |
| pnpm                  | โปรแกรมติดตั้งชุดคำสั่งและไลบรารีของเว็บ           |
| VS Code               | โปรแกรมเปิดและแก้ไขไฟล์โค้ด                        |
| `.env`                | ไฟล์เก็บค่าลับ เช่น URL และ key สำหรับต่อฐานข้อมูล |

## สิ่งที่ต้องเตรียม

### 1. เครื่องคอมพิวเตอร์

แนะนำ

- Windows 10 หรือ Windows 11
- RAM อย่างน้อย 8 GB
- พื้นที่ว่างอย่างน้อย 5 GB
- อินเทอร์เน็ตเสถียร
- Browser: Chrome หรือ Microsoft Edge

ถ้าเครื่องช้าหรือเน็ตหลุดบ่อย ให้ทำทีละขั้น อย่าเปิดหลายโปรแกรมพร้อมกัน

### 2. บัญชีที่ต้องมี

| บัญชี    | ใช้ทำอะไร          | สมัครที่               |
| -------- | ------------------ | ---------------------- |
| GitHub   | ดาวน์โหลดโค้ด      | `https://github.com`   |
| Supabase | สร้างฐานข้อมูล     | `https://supabase.com` |
| Netlify  | เอาเว็บขึ้นออนไลน์ | `https://netlify.com`  |

คำแนะนำ: ใช้อีเมลของหน่วยงานถ้ามี เพื่อให้ส่งต่องานให้ทีมได้ง่าย

### 3. โปรแกรมที่ต้องติดตั้ง

| โปรแกรม     | ใช้ทำอะไร            | วิธีตรวจว่ามีแล้วหรือยัง     |
| ----------- | -------------------- | ---------------------------- |
| Node.js LTS | รันเว็บ              | `node -v`                    |
| pnpm        | ติดตั้ง dependencies | `pnpm -v`                    |
| Git         | ดาวน์โหลดโค้ด        | `git --version`              |
| VS Code     | เปิดโปรเจกต์         | เปิดโปรแกรมได้จาก Start Menu |

## ขั้นที่ 1: เปิด PowerShell

บน Windows ให้ทำตามนี้

1. กดปุ่ม Start
2. พิมพ์ `PowerShell`
3. คลิก `Windows PowerShell`
4. จะเห็นหน้าต่างสีน้ำเงินหรือดำสำหรับพิมพ์คำสั่ง

ทดสอบว่าใช้คำสั่งได้

```powershell
pwd
```

ถ้าขึ้น path เช่น `C:\Users\ชื่อผู้ใช้` แปลว่าเปิดถูกแล้ว

## ขั้นที่ 2: ตรวจโปรแกรมที่ติดตั้ง

พิมพ์คำสั่งทีละบรรทัด

```powershell
node -v
```

ผลที่ควรเห็น

```text
v20.x.x
```

หรือ

```text
v22.x.x
```

ต่อด้วย

```powershell
git --version
```

ผลที่ควรเห็น

```text
git version 2.x.x
```

ต่อด้วย

```powershell
pnpm -v
```

ผลที่ควรเห็น

```text
9.x.x
```

ถ้า `pnpm` ยังไม่มี ให้ติดตั้งด้วยคำสั่ง

```powershell
npm install -g pnpm
```

แล้วปิด PowerShell เปิดใหม่ จากนั้นตรวจอีกครั้ง

```powershell
pnpm -v
```

```text
D:\projects
```

```powershell
mkdir C:\projects
cd C:\projects
```

## ขั้นที่ 6: ติดตั้ง dependencies

คำว่า dependencies หมายถึงชุดโปรแกรมย่อยที่เว็บต้องใช้ ถ้าไม่ติดตั้ง เว็บจะรันไม่ได้

ใน PowerShell ที่อยู่ในโฟลเดอร์ `npt_dashboard` ให้พิมพ์


ถ้ามี warning แต่ไม่มี error ส่วนใหญ่ยังไปต่อได้

## ขั้นที่ 8: สร้าง Supabase Project

เปิดเว็บ

```text
https://supabase.com
```

ทำตามนี้

1. กด `Sign in`
2. เข้าสู่ระบบด้วยบัญชีของหน่วยงานหรือบัญชีที่เตรียมไว้
3. กด `New project`
4. เลือก Organization
5. ตั้งชื่อ project เช่น `npt-smart-agri-dashboard`
6. ตั้งรหัสผ่านฐานข้อมูล เก็บไว้ให้ดี
7. เลือก Region ใกล้ประเทศไทย เช่น Singapore
8. กด `Create new project`
9. รอจน project สร้างเสร็จ

สิ่งที่ต้องจดไว้

- Project URL
- anon public key
- database password

อย่าส่ง service role key ให้คนทั่วไป และอย่าใส่ service role key ใน frontend

## ขั้นที่ 9: คัดลอก Project URL และ anon key

ใน Supabase project

1. ไปที่ `Project Settings`
2. ไปที่ `API`
3. คัดลอก `Project URL`
4. คัดลอก `anon public key`

ตัวอย่างค่า

```text
Project URL:
https://abcdefghijk.supabase.co

anon public key:
คีย์ยาวจาก Supabase ที่ขึ้นต้นด้วยตัวอักษรหลายตัว
```

## ขั้นที่ 10: สร้างไฟล์ `.env`

กลับมาที่ VS Code

1. ดูไฟล์ `.env.example`
2. สร้างไฟล์ใหม่ชื่อ `.env`
3. ใส่ค่าตามตัวอย่างนี้

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

ให้แทนค่า

- `https://your-project.supabase.co` ด้วย Project URL จริง
- `your-anon-key-here` ด้วย anon public key จริง

ตัวอย่าง

```env
VITE_SUPABASE_URL=https://abcdefghijk.supabase.co
VITE_SUPABASE_ANON_KEY=คัดลอก-anon-public-key-จาก-supabase-มาวางตรงนี้
```

ข้อสำคัญ

- ห้ามมีช่องว่างก่อนหรือหลังเครื่องหมาย `=`
- ห้ามใส่เครื่องหมายคำพูดถ้าไม่จำเป็น
- ห้าม commit ไฟล์ `.env`
- repo นี้ตั้งค่า `.gitignore` ไว้แล้ว แต่ต้องตรวจทุกครั้งก่อนส่งงาน

ตรวจว่า Git ไม่เห็นไฟล์ `.env`

```powershell
git status
```

ไม่ควรเห็น `.env` ในรายการไฟล์ที่จะ commit

## ขั้นที่ 11: สร้างตารางฐานข้อมูลใน Supabase

โปรเจกต์นี้มีไฟล์ SQL อยู่ในโฟลเดอร์ `supabase/`

ให้เปิดดูใน VS Code

```text
supabase/
```

แนวทางทั่วไป

1. เปิด Supabase project
2. เข้าเมนู `SQL Editor`
3. กด `New query`
4. คัดลอก SQL จากไฟล์ schema หรือ migration ที่ทีมกำหนด
5. วางใน SQL Editor
6. กด `Run`
7. ตรวจว่ามีตารางถูกสร้างในเมนู `Table Editor`

ถ้าไม่แน่ใจว่าต้องรันไฟล์ไหน ให้เริ่มจากอ่านบทนี้ก่อน

- `04-การออกแบบฐานข้อมูลและตั้งค่า-supabase.md`

คำเตือน: อย่ารัน SQL ที่มีคำว่า `DROP TABLE`, `DELETE FROM` หรือ `TRUNCATE` กับฐานข้อมูลจริง ถ้าไม่มั่นใจ

## ขั้นที่ 12: สร้างผู้ใช้สำหรับทดสอบ Login

ใน Supabase

1. ไปที่ `Authentication`
2. ไปที่ `Users`
3. กด `Add user`
4. ใส่อีเมลและรหัสผ่าน
5. ถ้ามีตัวเลือกยืนยันอีเมล ให้ตั้งตามนโยบายของหน่วยงาน
6. บันทึก

หลังจากสร้าง user แล้ว ต้องมีข้อมูลในตาราง `profiles` ด้วย เพื่อให้ระบบรู้ว่า user นี้มี role อะไร

ตัวอย่าง role

| role     | ใช้ทำอะไร                  |
| -------- | -------------------------- |
| `admin`  | ดูแลระบบทั้งหมด            |
| `editor` | เพิ่ม/แก้ไขข้อมูลตามสิทธิ์ |
| `viewer` | เปิดดูข้อมูล               |

ถ้า login ได้แต่ไม่เห็นเมนู ให้ตรวจตาราง `profiles` ก่อน

## ขั้นที่ 14: ตรวจ build ก่อนส่งเว็บขึ้นออนไลน์

ก่อน deploy ต้องตรวจว่า build ผ่าน

หยุด dev server ก่อนด้วย `Ctrl + C`

จากนั้นพิมพ์

```powershell
pnpm build
```

ผลที่ควรเห็น

- ไม่มี error
- มีโฟลเดอร์ `dist`
- ขึ้นข้อความ build completed หรือ done

ถ้า build fail ให้ดูบรรทัดแรกที่เป็น error จริง ไม่ต้องตกใจข้อความยาว ๆ ทั้งหมด

ทดสอบไฟล์ build ในเครื่อง

```powershell
pnpm preview
```

เปิดลิงก์ที่ขึ้น เช่น

```text
http://localhost:4173/
```

ตรวจว่าเว็บยังเปิดได้

## ขั้นที่ 15: เตรียม Netlify

เปิด

```text
https://netlify.com
```

เข้าสู่ระบบ แล้วทำตามนี้

1. กด `Add new site`
2. เลือก `Import an existing project`
3. เลือก GitHub
4. อนุญาตให้ Netlify เข้าถึง repository
5. เลือก repo `npt_dashboard`

ตั้งค่า build

| ช่อง              | ค่า          |
| ----------------- | ------------ |
| Build command     | `pnpm build` |
| Publish directory | `dist`       |

ถ้า Netlify ถาม Node version ให้ใช้ LTS เช่น 20 หรือ 22

## ขั้นที่ 16: ใส่ Environment Variables บน Netlify

ใน Netlify site

1. ไปที่ `Site configuration`
2. ไปที่ `Environment variables`
3. กด `Add a variable`
4. เพิ่มค่าเหล่านี้

```text
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```

ค่าให้ตรงกับไฟล์ `.env` ในเครื่อง

ถ้าใช้ AI Chatbot ผ่าน Netlify Function ให้เพิ่ม key ที่ระบบใช้จริง เช่น

```text
GEMINI_API_KEY
OPENROUTER_API_KEY
NVIDIA_API_KEY
```

ใส่เฉพาะ key ที่ใช้จริง อย่าใส่ key ทดสอบมั่ว ๆ

## ขั้นที่ 17: Deploy เว็บ

ใน Netlify

1. กด `Deploy site`
2. รอ build
3. ถ้าสำเร็จ จะได้ URL เช่น

```text
https://npt-smart-agri-dashboard.netlify.app
```

เปิด URL นั้นเพื่อตรวจเว็บจริง

## ขั้นที่ 18: ตรวจหลังเว็บออนไลน์

เปิดเว็บจาก URL ของ Netlify แล้วตรวจตามนี้

- [ ] หน้าแรกเปิดได้
- [ ] refresh หน้าแล้วไม่ขึ้น 404
- [ ] หน้า login เปิดได้
- [ ] login ได้
- [ ] dashboard เปิดได้
- [ ] ข้อมูลจาก Supabase แสดงผล
- [ ] search ใช้งานได้
- [ ] chatbot ใช้งานได้ถ้าตั้งค่า AI แล้ว
- [ ] เปิดบนมือถือได้
- [ ] ไม่มีข้อมูลส่วนบุคคลที่ไม่ควรเผยแพร่บนหน้า public

ถ้า refresh แล้ว 404 ให้ตรวจ `netlify.toml` ว่ามี redirect นี้

```toml
[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200
```

## ขั้นที่ 19: วิธีอัปเดตข้อมูลรอบถัดไป
