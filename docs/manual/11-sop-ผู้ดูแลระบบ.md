# 11 SOP ผู้ดูแลระบบ

SOP นี้เป็นรายการงานสั้นสำหรับผู้ดูแลระบบหลังเปิดใช้งาน `NPT Smart Agri Dashboard` แล้ว ใช้เช็กประจำวัน ประจำสัปดาห์ ประจำเดือน และใช้รับมือเมื่อมีปัญหา

## บทบาทผู้ดูแลระบบ

ผู้ดูแลระบบต้องดูแล 5 เรื่อง:

1. ผู้ใช้และสิทธิ์
2. คุณภาพข้อมูล
3. ความปลอดภัยของข้อมูล
4. สถานะเว็บและการ deploy
5. การสำรองข้อมูลและบันทึกเหตุการณ์

## งานรายวัน

ใช้เวลา 5-10 นาที:

- [ ] เปิดหน้าเว็บจริงได้
- [ ] Login ด้วยบัญชีทดสอบได้
- [ ] Dashboard หลักโหลดข้อมูลได้
- [ ] Search ใช้งานได้
- [ ] AI Chatbot ตอบได้ ถ้าเปิดใช้งาน
- [ ] ไม่มีข้อมูลส่วนบุคคลโผล่ในหน้า public
- [ ] ไม่มีผู้ใช้แจ้งปัญหาเร่งด่วนค้างอยู่

ถ้าพบปัญหา ให้บันทึกในแบบฟอร์ม incident ด้านล่าง

## งานรายสัปดาห์

- [ ] ตรวจข้อมูลใหม่ที่เพิ่มเข้าระบบ
- [ ] ตรวจรายการซ้ำ
- [ ] ตรวจอำเภอ/ตำบลสะกดผิด
- [ ] ตรวจข้อมูลตัวเลขผิดปกติ
- [ ] ตรวจ Netlify deploy logs
- [ ] ตรวจ Supabase usage เบื้องต้น
- [ ] export backup ตารางสำคัญ
- [ ] ทบทวนปัญหาที่ผู้ใช้แจ้งในสัปดาห์นั้น

## งานรายเดือน

- [ ] ตรวจรายชื่อผู้ใช้ทั้งหมด
- [ ] ปิดสิทธิ์ผู้ใช้ที่ย้ายงานหรือไม่เกี่ยวข้องแล้ว
- [ ] ตรวจ role ของ admin/editor/viewer
- [ ] ตรวจ audit log หรือประวัติการแก้ไข
- [ ] ทดสอบ backup ว่าเปิดอ่านได้
- [ ] ทบทวน environment variables บน Netlify
- [ ] ตรวจ package/dependency เฉพาะเมื่อมีแผนอัปเดต
- [ ] สรุปรายการปัญหาและสิ่งที่ควรปรับปรุง

## วิธีเพิ่มผู้ใช้

1. เข้า Supabase
2. ไปที่ `Authentication -> Users`
3. เพิ่มผู้ใช้หรือเชิญผู้ใช้ตามนโยบายของหน่วยงาน
4. ไปที่ตาราง `profiles`
5. เพิ่มหรือแก้ row ของผู้ใช้นั้น
6. ตั้งค่า `email`, `full_name`, `department`, `role`
7. ให้ผู้ใช้ลอง login
8. ตรวจว่าเห็นเมนูตามสิทธิ์

role ที่ใช้:

| role     | ใช้กับใคร                 |
| -------- | ------------------------- |
| `viewer` | ดูข้อมูล                  |
| `editor` | เพิ่ม/แก้ไขข้อมูล         |
| `admin`  | ดูแลระบบ ผู้ใช้ และสิทธิ์ |

ห้ามให้ `admin` ถ้าไม่จำเป็น

## วิธีปิดสิทธิ์ผู้ใช้

ใช้เมื่อเจ้าหน้าที่พ้นหน้าที่ ย้ายงาน หรือไม่ควรเข้าระบบแล้ว

1. เข้า Supabase
2. ไปที่ `Authentication -> Users`
3. ปิดหรือระงับบัญชีตามวิธีที่หน่วยงานใช้
4. ไปที่ตาราง `profiles`
5. เปลี่ยน role เป็นค่าว่างหรือ role ที่ไม่มีสิทธิ์ใช้งาน
6. บันทึกวันที่และเหตุผล
7. ทดสอบว่าบัญชีนั้นเข้าเมนูภายในไม่ได้

Checklist ปิดสิทธิ์:

- [ ] ทราบเหตุผลการปิดสิทธิ์
- [ ] ปิดใน Auth แล้ว
- [ ] แก้ `profiles` แล้ว
- [ ] ตรวจว่า login หรือเข้าเมนูไม่ได้แล้ว
- [ ] บันทึกผู้ดำเนินการและวันที่แล้ว

## วิธีเปลี่ยน role

1. ตรวจคำขอจากหัวหน้างาน
2. เปิด Supabase
3. เข้า table `profiles`
4. หา email ผู้ใช้
5. แก้ `role`
6. กด save
7. ให้ผู้ใช้ออกจากระบบแล้ว login ใหม่
8. ตรวจเมนูที่เห็น

ถ้า login ได้แต่ไม่มีเมนู ให้ตรวจ `profiles.email`, `profiles.role` และ `profiles.department`

## วิธี backup ข้อมูล

ก่อนแก้ข้อมูลจำนวนมากหรือ deploy ใหญ่ ให้ backup ก่อน

ตารางที่ควร backup:

- `profiles`
- ตารางข้อมูลหลักที่แสดงบน Dashboard
- ตาราง Data Request ถ้ามี
- ตาราง audit log ถ้ามี

รูปแบบชื่อไฟล์:

```text
table_name_backup_YYYY-MM-DD.csv
```

ตัวอย่าง:

```text
profiles_backup_2026-07-02.csv
large_plots_backup_2026-07-02.csv
```

หลัง backup ให้เปิดไฟล์ตรวจว่าอ่านได้และภาษาไทยไม่เพี้ยน

## ก่อนแก้ข้อมูลจำนวนมาก

- [ ] ระบุว่าจะแก้ตารางใด
- [ ] ระบุจำนวนรายการโดยประมาณ
- [ ] backup ตารางนั้นแล้ว
- [ ] มีไฟล์ clean หรือ SQL ที่ตรวจแล้ว
- [ ] มีคนรับผิดชอบอนุมัติ
- [ ] รู้วิธีย้อนกลับถ้าแก้ผิด

ถ้ายังไม่มี backup ห้ามแก้ข้อมูลจำนวนมาก

## วิธีรับแจ้งปัญหา

เมื่อผู้ใช้แจ้งปัญหา ให้ขอข้อมูลนี้:

```text
ชื่อผู้แจ้ง:
วันเวลา:
หน้าเว็บที่ใช้งาน:
ทำอะไรอยู่:
คาดว่าจะเห็นอะไร:
เห็นปัญหาอะไร:
ข้อความ error:
รูปหน้าจอ:
บัญชีผู้ใช้/role:
```

จัดระดับปัญหา:

| ระดับ | ตัวอย่าง                                   | เวลาตอบสนอง      |
| ----- | ------------------------------------------ | ---------------- |
| สูง   | Login ไม่ได้ทั้งระบบ, ข้อมูลลับโผล่ public | ทันที            |
| กลาง  | Import ไม่ได้, Dashboard บางส่วนไม่ขึ้น    | ภายในวันทำการ    |
| ต่ำ   | คำผิด, ข้อเสนอแนะ, ปรับข้อความ             | รอบปรับปรุงถัดไป |

## แบบฟอร์ม incident

```text
เลขที่ incident:
วันเวลาเริ่ม:
ผู้แจ้ง:
ผู้รับผิดชอบ:
ระดับปัญหา:
อาการ:
ผลกระทบ:
สาเหตุที่พบ:
วิธีแก้:
วันเวลาที่แก้เสร็จ:
ต้องติดตามต่อหรือไม่:
```

## ตรวจ Netlify

ตรวจเมื่อเว็บเปิดไม่ได้หรือ deploy มีปัญหา:

1. เข้า Netlify
2. เปิด site ของระบบ
3. ไปที่ `Deploys`
4. ดู deploy ล่าสุด
5. ถ้า fail ให้เปิด log
6. ดูบรรทัดที่มีคำว่า `error`
7. ถ้าแก้ไม่ได้ ให้ส่ง log ให้ทีมเทคนิค

ถ้า deploy ใหม่แล้วเว็บพัง ให้ rollback ไป deploy ก่อนหน้าที่ใช้งานได้

## ตรวจ Supabase

ตรวจเมื่อข้อมูลไม่ขึ้นหรือ login มีปัญหา:

- URL/key ใน `.env` หรือ Netlify env ตรง project จริงหรือไม่
- ตารางมีข้อมูลหรือไม่
- `profiles` มี row ของผู้ใช้หรือไม่
- role ถูกต้องหรือไม่
- RLS อนุญาตให้อ่านหรือเขียนหรือไม่

## สิ่งที่ห้ามทำ

- ห้ามใช้ service role key ใน frontend
- ห้าม commit `.env` หรือ key จริงเข้า repo
- ห้ามลบข้อมูล production ถ้ายังไม่มี backup
- ห้ามให้สิทธิ์ admin เพราะสะดวก
- ห้ามเอาไฟล์ที่มีข้อมูลส่วนบุคคลไปวาง public
- ห้ามแก้ production แบบลองผิดลองถูกหลายรอบ
