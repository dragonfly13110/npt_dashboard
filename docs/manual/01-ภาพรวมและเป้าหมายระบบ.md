# 01 ภาพรวมและเป้าหมายระบบ

ไฟล์นี้ใช้วางภาพรวมของระบบศูนย์ข้อมูลการเกษตรจังหวัด ว่าระบบนี้สร้างขึ้นเพื่ออะไร ใช้กับใคร และควรครอบคลุมงานส่วนใดบ้าง

## 1. เป้าหมายของระบบ

ระบบ `npt_dashboard` มีเป้าหมายเพื่อรวบรวมข้อมูลสำคัญด้านการเกษตรของจังหวัดไว้ในระบบเดียว และนำเสนอในรูปแบบที่เจ้าหน้าที่ ผู้บริหาร และประชาชนสามารถเข้าถึงได้ตามสิทธิ์ที่เหมาะสม

เป้าหมายหลัก ได้แก่

- รวมข้อมูลจากหลายกลุ่มงานไว้ในฐานข้อมูลกลาง
- ลดการค้นหาไฟล์ Excel หลายชุด
- แสดงข้อมูลสำคัญเป็น Dashboard และแผนที่
- เปิดเผยข้อมูลบางส่วนให้ประชาชนเข้าถึงได้
- ช่วยเจ้าหน้าที่ค้นหา วิเคราะห์ และสรุปข้อมูลได้เร็วขึ้น
- ใช้ AI Chatbot ช่วยถามตอบจากฐานข้อมูลที่มีอยู่

## 2. ปัญหาที่ระบบต้องการแก้

- ข้อมูลกระจายอยู่หลายไฟล์ หลายกลุ่มงาน และหลายรูปแบบ
- ชื่อคอลัมน์และรูปแบบข้อมูลไม่สม่ำเสมอ
- การทำรายงานต้องใช้เวลารวบรวมข้อมูลซ้ำ ๆ
- ข้อมูลแผนที่และข้อมูลตารางไม่ได้เชื่อมกัน
- ผู้บริหารต้องการดูภาพรวม แต่ข้อมูลอยู่ในรูปแบบที่อ่านยาก
- เจ้าหน้าที่ต้องตอบคำถามจากข้อมูลเดิมบ่อยครั้ง

## 3. กลุ่มผู้ใช้งาน

| กลุ่มผู้ใช้ | บทบาท |
|---|---|
| Public / Guest | ดูข้อมูลสาธารณะ เช่น แปลงใหญ่ วิสาหกิจชุมชน ท่องเที่ยวเกษตร และ Dashboard บางส่วน |
| Viewer | ดูข้อมูลภายในตามสิทธิ์ |
| Editor | ดูและแก้ไขข้อมูลในกลุ่มงานที่รับผิดชอบ |
| Admin | จัดการข้อมูล ผู้ใช้ สิทธิ์ และระบบโดยรวม |
| ผู้บริหาร | ดู Dashboard สรุปภาพรวมและข้อมูลประกอบการตัดสินใจ |

## 4. ขอบเขตระบบ

ระบบควรครอบคลุมข้อมูลหลักอย่างน้อย 5 กลุ่ม

1. ข้อมูลยุทธศาสตร์และสารสนเทศ
2. ข้อมูลส่งเสริมและพัฒนาการผลิต
3. ข้อมูลส่งเสริมและพัฒนาเกษตรกร
4. ข้อมูลอารักขาพืช
5. ข้อมูลบริหารทั่วไป

## 5. โมดูลหลักของระบบ

- Public Landing Page
- Interactive Dashboard
- Internal Dashboard
- Data Management
- Global Search
- AI Chatbot
- User Management
- Audit / Recent Activities
- Map and Visualization

## 6. หลักคิดเรื่องข้อมูลสาธารณะและข้อมูลภายใน

ก่อนนำข้อมูลเข้า Dashboard ต้องแยกให้ชัดว่า

- ข้อมูลใดเผยแพร่สาธารณะได้
- ข้อมูลใดให้เฉพาะเจ้าหน้าที่ดู
- ข้อมูลใดเป็นข้อมูลส่วนบุคคล
- ข้อมูลใดต้องสรุปเป็นภาพรวมก่อนเผยแพร่
- ข้อมูลใดต้องขออนุญาตเจ้าของข้อมูลก่อนใช้

## 7. Checklist ก่อนเริ่มโครงการ

- [ ] ระบุเจ้าของโครงการแล้ว
- [ ] ระบุผู้ดูแลระบบแล้ว
- [ ] ระบุกลุ่มงานเจ้าของข้อมูลแล้ว
- [ ] ระบุข้อมูลที่ต้องใช้ใน Dashboard แล้ว
- [ ] ระบุข้อมูล public และ internal แล้ว
- [ ] ระบุระดับสิทธิ์ผู้ใช้แล้ว
- [ ] มีแผนการอัปเดตข้อมูลหลังเปิดใช้งานแล้ว

## 8. ผลลัพธ์ที่ควรได้จากขั้นตอนนี้

หลังจบบทนี้ ควรได้เอกสารสั้น ๆ ที่ตอบได้ว่า

- ระบบนี้ทำเพื่ออะไร
- ใครเป็นผู้ใช้งานหลัก
- ใช้ข้อมูลอะไรบ้าง
- ข้อมูลใดเผยแพร่ได้
- ข้อมูลใดต้องจำกัดสิทธิ์
- ใครเป็นผู้รับผิดชอบแต่ละชุดข้อมูล
