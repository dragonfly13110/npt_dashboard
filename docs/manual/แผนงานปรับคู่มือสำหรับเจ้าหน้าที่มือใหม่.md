# แผนงานปรับคู่มือสำหรับเจ้าหน้าที่มือใหม่

เอกสารนี้เป็นแผนทำงานสำหรับยกระดับคู่มือ `NPT Smart Agri Dashboard` ให้เหมาะกับเจ้าหน้าที่ นักวิชาการเกษตร และผู้ดูแลข้อมูลที่อาจไม่ถนัดคอมพิวเตอร์ แต่อยากศึกษาและทำตามจนติดตั้งระบบได้จริง

## เป้าหมายของงาน

ทำให้คู่มือเปลี่ยนจาก "เอกสารอธิบายระบบ" เป็น "คู่มือปฏิบัติงานจริง" ที่ผู้อ่านสามารถทำตามได้ตั้งแต่ต้นจนจบ

ผลลัพธ์ที่ต้องการ

- เจ้าหน้าที่เปิดอ่านแล้วรู้ทันทีว่าต้องเริ่มตรงไหน
- ทำตามแล้วเปิดเว็บในเครื่องได้
- เชื่อม Supabase ได้
- เข้าใจการเตรียมข้อมูลเบื้องต้น
- Deploy ขึ้น Netlify ได้
- แก้ปัญหาพื้นฐานเองได้
- รู้ว่าข้อมูลใดห้ามเผยแพร่
- มี checklist สำหรับส่งมอบและดูแลต่อ

## หลักการเขียน

1. ใช้ภาษาคนทำงานราชการ ไม่ใช้ภาษานักพัฒนาเป็นหลัก
2. ทุกคำเทคนิคต้องมีคำแปล
3. ทุกขั้นตอนต้องบอกว่าต้องคลิกอะไร พิมพ์อะไร และควรเห็นอะไร
4. ห้ามข้ามขั้นตอนที่คนทำเป็นมองว่าเล็ก เช่น เปิด PowerShell, เข้าโฟลเดอร์, คัดลอก key
5. ทุกบทต้องมี checklist
6. ทุกบทที่เสี่ยงต้องมีคำเตือน
7. แยก "ต้องทำ" ออกจาก "อ่านเพิ่ม"
8. ถ้าต้องใช้คำสั่ง ต้องใส่คำสั่งเต็ม ไม่ให้ผู้อ่านเดา

## กลุ่มผู้อ่าน

งานคู่มือชุดนี้ถือว่าพร้อม เมื่อผ่านเงื่อนไขเหล่านี้

- [ ] คนที่ไม่เคยทำเว็บอ่านบทเริ่มต้นแล้วเปิดเว็บ local ได้
- [ ] คนที่ไม่เคยใช้ Supabase สร้าง project และคัดลอก key ได้
- [ ] คนที่ไม่เคย Deploy เว็บส่งขึ้น Netlify ได้
- [ ] มี troubleshooting สำหรับ error หลัก
- [ ] มี checklist ก่อนเปิดใช้งานจริง
- [ ] มีคำเตือนข้อมูลส่วนบุคคลชัดเจน
- [ ] มี SOP ดูแลระบบต่อ
- [ ] คู่มือทั้งหมดถูกเชื่อมใน `docs/manual/README.md`
- [ ] ไม่มี secret หรือ key จริงในเอกสาร
- [ ] ภาษาไทยอ่านไม่เพี้ยนใน editor และบนเว็บ
