# 10 คู่มือนำเข้าข้อมูล CSV สำหรับเจ้าหน้าที่

คู่มือนี้ใช้เมื่อต้องนำข้อมูลจาก Excel หรือ CSV เข้า `NPT Smart Agri Dashboard` โดยไม่ต้องแก้โค้ด เหมาะกับเจ้าหน้าที่ข้อมูลที่ทำงานกับไฟล์ตารางเป็นหลัก

## ก่อนเริ่ม

ต้องมีสิ่งเหล่านี้ก่อน:

- ไฟล์ต้นฉบับจากเจ้าของข้อมูล
- ไฟล์สำรองที่ยังไม่แก้ไข
- ไฟล์ที่ clean แล้วสำหรับนำเข้า
- บัญชีผู้ใช้ที่มีสิทธิ์เพิ่มหรือแก้ไขข้อมูล
- ชื่อตารางเป้าหมาย เช่น แปลงใหญ่ ศูนย์เรียนรู้ หรือข้อมูลภัยพิบัติ

ห้ามนำไฟล์ที่มีเลขบัตรประชาชน เบอร์โทร ที่อยู่ละเอียด หรือข้อมูลส่วนบุคคลขึ้นหน้า public โดยไม่ได้รับอนุญาต

## 1. เก็บไฟล์ต้นฉบับไว้ก่อน

1. สร้างโฟลเดอร์ `raw`
2. คัดลอกไฟล์ที่ได้รับมาไว้ใน `raw`
3. ห้ามแก้ไฟล์ใน `raw`
4. สร้างไฟล์ทำงานอีกชุดในโฟลเดอร์ `working`

ตัวอย่าง:

```text
data/
  raw/large_plots_original_2569.xlsx
  working/large_plots_clean_2569.xlsx
  ready_to_import/large_plots_2569.csv
```

ถ้าทำถูก: ยังมีไฟล์ต้นฉบับให้ย้อนกลับได้เสมอ

## 2. ทำตารางให้เป็นรูปแบบปกติ

ไฟล์พร้อมนำเข้าควรมีลักษณะนี้:

- แถวแรกเป็นหัวคอลัมน์
- หนึ่งแถวคือหนึ่งรายการ
- หนึ่งคอลัมน์คือหนึ่งชนิดข้อมูล
- ไม่มี merged cells
- ไม่มีหัวรายงานหลายบรรทัด
- ไม่มีแถวรวมยอดปนกับข้อมูล
- ไม่มีรูปภาพ กล่องข้อความ หรือหมายเหตุยาวในตารางหลัก

ตัวอย่างที่ควรแก้:

| ปัญหา                      | วิธีแก้                                      |
| -------------------------- | -------------------------------------------- |
| มีหัวรายงาน 3 แถวก่อนตาราง | ลบออกจากไฟล์นำเข้า หรือย้ายไป sheet หมายเหตุ |
| รวม cell ชื่ออำเภอ         | เติมชื่ออำเภอซ้ำทุกแถว                       |
| มีแถวรวมยอดท้ายตาราง       | ลบออกจากไฟล์นำเข้า                           |
| พื้นที่เขียนว่า `25 ไร่`   | แยกเป็น `area_rai = 25`                      |
| วันที่มีหลายรูปแบบ         | แปลงเป็น `YYYY-MM-DD`                        |

## 3. ตั้งชื่อคอลัมน์

ถ้าต้องการให้ระบบ map ง่าย ให้ใช้ชื่อภาษาอังกฤษแบบ `snake_case`

| ชื่อควรใช้    | ไม่ควรใช้          |
| ------------- | ------------------ |
| `district`    | `อำเภอ `           |
| `subdistrict` | `ตำบล/แขวง`        |
| `crop_name`   | `ชนิดพืช (ระบุ)`   |
| `area_rai`    | `พื้นที่รวม (ไร่)` |
| `data_year`   | `ปีงบประมาณ`       |

ถ้ายังต้องใช้หัวคอลัมน์ภาษาไทย ให้ทำตาราง mapping ไว้ก่อนนำเข้า

| คอลัมน์ในไฟล์ | field ในระบบ  |
| ------------- | ------------- |
| อำเภอ         | `district`    |
| ตำบล          | `subdistrict` |
| ชนิดพืช       | `crop_name`   |
| พื้นที่ไร่    | `area_rai`    |

## 4. ตรวจข้อมูลสำคัญ

ตรวจอย่างน้อย 5 จุดนี้:

1. อำเภอและตำบลสะกดตรงกันทุกแถว
2. ตัวเลขเป็นตัวเลขจริง ไม่มีหน่วยปน
3. วันที่อยู่รูปแบบเดียวกัน
4. ไม่มีรายการซ้ำโดยไม่ตั้งใจ
5. ข้อมูลส่วนบุคคลถูกแยกออกหรือจำกัดสิทธิ์แล้ว

ตัวอย่างรายการซ้ำที่ต้องดู:

| ตาราง         | วิธีดูซ้ำ                          |
| ------------- | ---------------------------------- |
| แปลงใหญ่      | ชื่อแปลง + อำเภอ + ปีข้อมูล        |
| ศูนย์เรียนรู้ | ชื่อศูนย์ + อำเภอ                  |
| จุดแผนที่     | ชื่อสถานที่ + latitude + longitude |

## 5. Export เป็น CSV

ใน Excel:

1. เปิดไฟล์ที่ clean แล้ว
2. กด `File`
3. กด `Save As`
4. เลือกชนิดไฟล์ `CSV UTF-8 (Comma delimited) (*.csv)`
5. ตั้งชื่อไฟล์ให้บอกตารางและปีข้อมูล

ตัวอย่างชื่อไฟล์:

```text
large_plots_2569_ready.csv
learning_centers_2569_ready.csv
disaster_points_2569_ready.csv
```

ถ้าทำถูก: เปิดไฟล์ CSV แล้วภาษาไทยไม่เพี้ยน

## 6. Backup ก่อนนำเข้า

ก่อน import ทุกครั้ง ให้ export ข้อมูลเดิมจากระบบเก็บไว้ก่อน

ชื่อไฟล์ backup ควรมีวันที่:

```text
large_plots_backup_2026-07-02.csv
```

ห้าม import ทับข้อมูลจำนวนมากถ้ายังไม่มี backup

## 7. นำเข้าในหน้า Data Management

1. Login เข้าระบบ
2. เปิดหน้า Dashboard
3. เข้าเมนู Data Management หรือตารางที่ต้องการ
4. เลือกตารางเป้าหมาย
5. กด import CSV
6. เลือกไฟล์ CSV ที่เตรียมไว้
7. ตรวจ mapping คอลัมน์
8. กดนำเข้า
9. รอข้อความสำเร็จ

ถ้าระบบถามว่าจะเพิ่มข้อมูลใหม่หรืออัปเดตข้อมูลเดิม ให้เลือกตามงานจริง อย่าเดา ถ้าไม่แน่ใจให้หยุดและถามผู้ดูแลระบบ

## 8. ตรวจหลังนำเข้า

หลัง import ให้ตรวจทันที:

- [ ] จำนวนแถวในระบบใกล้เคียงกับไฟล์
- [ ] ข้อมูลตัวอย่าง 3-5 แถวถูกต้อง
- [ ] ค้นหาด้วยชื่อหรืออำเภอแล้วเจอ
- [ ] Dashboard แสดงตัวเลขใหม่
- [ ] Map แสดงตำแหน่งถูก ถ้ามีพิกัด
- [ ] หน้า public ไม่แสดงข้อมูลส่วนบุคคล
- [ ] Export กลับออกมาแล้วภาษาไทยไม่เพี้ยน

## 9. ถ้านำเข้าไม่สำเร็จ

| อาการ                    | ตรวจจุดนี้ก่อน                          |
| ------------------------ | --------------------------------------- |
| ภาษาไทยเพี้ยน            | export เป็น `CSV UTF-8` หรือยัง         |
| ระบบบอก column ไม่ตรง    | ตรวจชื่อคอลัมน์และ mapping              |
| ตัวเลขไม่เข้า            | มีหน่วยหรือ comma ปนในช่องตัวเลขหรือไม่ |
| วันที่ error             | ใช้รูปแบบ `YYYY-MM-DD` หรือยัง          |
| ข้อมูลไม่ขึ้นหลัง import | ตรวจสิทธิ์ RLS และตารางเป้าหมาย         |
| import แล้วข้อมูลซ้ำ     | ตรวจ key ที่ใช้เทียบรายการเดิม          |

ถ้ายังแก้ไม่ได้ ให้ส่งข้อมูลนี้ให้ทีมเทคนิค:

```text
ตารางที่นำเข้า:
ชื่อไฟล์:
ขั้นตอนที่ทำ:
ข้อความ error:
จำนวนแถวในไฟล์:
รูปหน้าจอ error:
```

## Checklist ก่อน import

- [ ] สำรองไฟล์ต้นฉบับแล้ว
- [ ] สร้างไฟล์ clean แยกจากต้นฉบับแล้ว
- [ ] ไม่มี merged cells
- [ ] ไม่มีแถวรวมยอด
- [ ] คอลัมน์สำคัญครบ
- [ ] ตรวจรายการซ้ำแล้ว
- [ ] แยกข้อมูลส่วนบุคคลแล้ว
- [ ] export เป็น CSV UTF-8 แล้ว
- [ ] backup ข้อมูลเดิมจากระบบแล้ว

## Checklist หลัง import

- [ ] ระบบแจ้งว่านำเข้าสำเร็จ
- [ ] จำนวนรายการถูกต้อง
- [ ] ค้นหาเจอ
- [ ] Dashboard/Map แสดงผลถูก
- [ ] ไม่มีข้อมูลลับโผล่ในหน้า public
- [ ] บันทึกชื่อไฟล์และวันที่นำเข้าแล้ว
