# 09 เทคนิคการสั่ง AI ช่วยทำงานทุกขั้นตอน

คู่มือนี้ใช้เปิดคู่กับคู่มือทุกบท เหมาะสำหรับคนที่ยังไม่เคยทำเว็บ ไม่เคยใช้ Supabase หรือไม่มั่นใจคำสั่งคอมพิวเตอร์ ให้ใช้ AI เป็นผู้ช่วยอธิบาย ตรวจงาน และแปล error แต่ไม่ให้ AI เดาสุ่มแทนเรา

## กติกาสำคัญก่อนใช้ AI

- ห้ามส่ง password, API key, service role key, token, `.env` จริง หรือข้อมูลส่วนบุคคลให้ AI
- ส่ง error ได้ แต่ลบ key, email ส่วนตัว, เบอร์โทร, เลขบัตรประชาชน และ URL ลับออกก่อน
- ให้ AI อธิบายทีละขั้น ไม่ให้ตอบยาวทีเดียวถ้าอ่านไม่ทัน
- ให้ AI บอกว่า "ถ้าทำถูกจะเห็นอะไร" ทุกครั้ง
- ถ้า AI ให้คำสั่งลบข้อมูล แก้ production หรือปิด security ให้หยุดถามผู้ดูแลระบบก่อน

## สูตร prompt พื้นฐาน

คัดลอก prompt นี้ไปใช้ได้เกือบทุกงาน:

```text
ฉันเป็นเจ้าหน้าที่มือใหม่ ไม่ถนัดคอมพิวเตอร์
กำลังทำขั้นตอน: [ใส่ชื่อขั้นตอน]
เป้าหมายคือ: [ใส่ผลลัพธ์ที่ต้องการ]
สิ่งที่เห็นบนหน้าจอตอนนี้คือ: [อธิบายหรือวาง error]

ช่วยบอกทีละขั้นว่า:
1. ต้องคลิกหรือพิมพ์อะไร
2. ถ้าทำถูกควรเห็นอะไร
3. ถ้าไม่สำเร็จให้ตรวจจุดไหนก่อน
4. มีข้อมูลลับอะไรที่ห้ามส่งหรือห้ามเผยแพร่ไหม

ตอบเป็นภาษาไทยง่าย ๆ และอย่าให้ฉันข้ามขั้นตอนเล็ก ๆ
```

## 1. ใช้ AI ก่อนเริ่มโครงการ

ใช้เมื่อยังไม่รู้ว่าต้องเตรียมอะไร

```text
ฉันต้องเริ่มทำ NPT Smart Agri Dashboard จากศูนย์
ฉันยังไม่เคยติดตั้งเว็บ ไม่เคยใช้ Supabase และไม่เคย deploy

ช่วยทำ checklist สิ่งที่ต้องเตรียมก่อนเริ่ม ได้แก่:
- โปรแกรมในเครื่อง
- บัญชีที่ต้องสมัคร
- ข้อมูลที่ต้องขอจากหน่วยงาน
- สิทธิ์หรือ key ที่ต้องระวัง

ขอแบบสั้น ใช้เช็กทีละข้อ
```

ใช้ผลลัพธ์คู่กับ:

- `00-เริ่มต้นสำหรับมือใหม่-ทำเว็บตั้งแต่ศูนย์จนออนไลน์.md`
- `01-ภาพรวมและเป้าหมายระบบ.md`

## 2. ใช้ AI ตอนติดตั้งโปรแกรม

ใช้เมื่อติดตั้ง Node.js, Git, pnpm หรือเปิด PowerShell ไม่เป็น

```text
ฉันใช้ Windows และกำลังติดตั้งเครื่องเพื่อรันเว็บ
ฉันต้องตรวจว่า Node.js, Git และ pnpm ใช้ได้หรือยัง

ช่วยบอกทีละขั้น:
1. เปิด PowerShell อย่างไร
2. ต้องพิมพ์คำสั่งอะไร
3. ถ้าสำเร็จจะเห็นหน้าตาประมาณไหน
4. ถ้าขึ้นว่า command not recognized ต้องแก้อย่างไร

ห้ามข้ามขั้นตอนเปิดโปรแกรม
```

ถ้ามี error ให้ใช้:

```text
นี่คือ error จาก PowerShell:
[วาง error]

ช่วยแปลว่าแปลว่าอะไร และให้ฉันแก้ทีละขั้น
อย่าให้คำสั่งที่ลบไฟล์หรือ reset เครื่อง
```

## 3. ใช้ AI ตอน clone repo และติดตั้ง dependencies

```text
ฉันกำลังดาวน์โหลดโค้ดจาก GitHub และติดตั้ง dependencies
คำสั่งที่ใช้คือ:
[วางคำสั่ง]

ผลลัพธ์ที่เห็นคือ:
[วางข้อความหน้าจอ]

ช่วยบอกว่า:
1. คำสั่งผ่านหรือไม่
2. ถ้าผ่าน ขั้นต่อไปคืออะไร
3. ถ้าไม่ผ่าน ต้องแก้จุดไหน
4. ต้องอยู่ในโฟลเดอร์ไหนก่อนรันคำสั่งต่อ
```

คำถามเช็กโฟลเดอร์:

```text
ฉันไม่แน่ใจว่าอยู่ในโฟลเดอร์โปรเจกต์ถูกไหม
ใน PowerShell ตอนนี้เห็น path เป็น:
[วาง path]

ช่วยบอกว่าต้องตรวจไฟล์อะไรเพื่อยืนยันว่าอยู่ในโปรเจกต์ `npt_dashboard`
```

## 4. ใช้ AI ตอนตั้งค่า `.env`

ใช้ช่วยเข้าใจ แต่ห้ามส่ง key จริง

```text
ฉันกำลังตั้งค่าไฟล์ .env สำหรับเชื่อม Supabase
ฉันรู้ว่าห้ามส่ง key จริงให้ AI

ช่วยอธิบายว่าไฟล์ .env ต้องมีค่าอะไรบ้าง
และแต่ละค่าหาได้จากหน้าไหนของ Supabase
ตอบโดยใช้ค่าตัวอย่างปลอมเท่านั้น
```

ถ้าเว็บไม่เห็นข้อมูล:

```text
เว็บเปิดได้ แต่ข้อมูลไม่ขึ้น
ฉันมีค่า VITE_SUPABASE_URL และ VITE_SUPABASE_ANON_KEY ใน .env แล้ว
ฉันจะไม่ส่ง key จริง

ช่วยทำ checklist ให้ตรวจว่า:
- URL ชี้ถูก project ไหม
- restart dev server หรือยัง
- ตารางมีข้อมูลไหม
- RLS บล็อกหรือไม่
```

## 5. ใช้ AI ตอนสร้าง Supabase

```text
ฉันไม่เคยใช้ Supabase
กำลังสร้าง project เพื่อใช้กับ dashboard

ช่วยอธิบายคำเหล่านี้แบบเทียบกับ Excel:
- table
- row
- column
- primary key
- Auth
- role
- RLS

แล้วบอกว่าขั้นตอนใดห้ามทำถ้าไม่แน่ใจ
```

ก่อนรัน SQL:

```text
ฉันมี SQL นี้และกำลังจะรันใน Supabase:
[วาง SQL โดยลบข้อมูลลับออก]

ช่วยตรวจแบบมือใหม่:
1. SQL นี้สร้าง/แก้/ลบอะไร
2. มีคำสั่งลบข้อมูลหรือเปลี่ยนสิทธิ์แรง ๆ ไหม
3. ก่อนรันต้อง backup อะไร
4. หลังรันต้องตรวจผลอย่างไร

ถ้ามี DROP, DELETE, TRUNCATE หรือ service role ให้เตือนชัด ๆ
```

## 6. ใช้ AI ตอนขอข้อมูลจากหน่วยงาน

```text
ฉันต้องขอข้อมูลจากอำเภอ/กลุ่มงาน เพื่อนำเข้า dashboard
ชุดข้อมูลที่ต้องการคือ: [ระบุชื่อข้อมูล]
ผู้ส่งข้อมูลคือ: [อำเภอ/กลุ่มงาน]

ช่วยร่างข้อความขอข้อมูลแบบราชการ อ่านง่าย
ระบุ:
- ขอข้อมูลอะไร
- รูปแบบไฟล์ที่ต้องการ
- คอลัมน์ที่ต้องมี
- วันส่งข้อมูล
- คำเตือนเรื่องข้อมูลส่วนบุคคล
```

ใช้ตรวจแบบฟอร์ม:

```text
นี่คือหัวคอลัมน์ในไฟล์ที่ได้รับ:
[วางชื่อคอลัมน์เท่านั้น ไม่ต้องวางข้อมูลส่วนบุคคล]

ช่วยตรวจว่าเพียงพอสำหรับทำ dashboard ไหม
ต้องเพิ่มคอลัมน์อะไร
คอลัมน์ไหนเสี่ยงข้อมูลส่วนบุคคล
```

## 7. ใช้ AI ตอน clean Excel/CSV

```text
ฉันมีไฟล์ Excel ที่จะนำเข้า dashboard
ปัญหาที่เห็นคือ:
[เช่น มี merged cells, วันที่หลายแบบ, แถวรวมยอด]

ช่วยบอกวิธี clean ทีละขั้นใน Excel:
1. ต้องแก้ตรงไหน
2. ตัวอย่างก่อนแก้/หลังแก้ควรเป็นอย่างไร
3. ต้อง export เป็น CSV UTF-8 อย่างไร
4. ต้องตรวจข้อมูลส่วนบุคคลตรงไหน
```

ตรวจหัวคอลัมน์:

```text
หัวคอลัมน์ในไฟล์คือ:
[วางเฉพาะหัวคอลัมน์]

ช่วยแนะนำชื่อ field แบบ snake_case สำหรับนำเข้าระบบ
ทำตาราง mapping ระหว่างหัวคอลัมน์เดิมกับชื่อ field ใหม่
```

## 8. ใช้ AI ตอน import CSV

```text
ฉันกำลัง import CSV เข้า dashboard
ตารางเป้าหมายคือ: [ชื่อตาราง]
จำนวนแถวในไฟล์คือ: [จำนวน]
คอลัมน์ในไฟล์คือ: [วางเฉพาะหัวคอลัมน์]

ช่วยทำ checklist ก่อน import:
- ต้อง backup อะไร
- ต้องตรวจ column อะไร
- ต้องดูข้อมูลซ้ำอย่างไร
- หลัง import ต้องตรวจหน้าไหน
```

ถ้า import fail:

```text
ฉัน import CSV แล้ว error:
[วาง error]

ข้อมูลที่บอกได้:
- ตารางเป้าหมาย: [ชื่อ]
- จำนวนแถว: [จำนวน]
- หัวคอลัมน์: [ชื่อคอลัมน์]

ช่วยแปล error และบอกวิธีแก้ทีละขั้น
ห้ามขอข้อมูลส่วนบุคคลหรือ key จริง
```

```text
ฉัน import ข้อมูลแล้ว ต้องตรวจว่า dashboard ใช้งานได้

- ตัวเลขรวม
- ตาราง
- search
- map
- export
- หน้า public
- สิทธิ์ผู้ใช้

ขอแบบที่เจ้าหน้าที่ติ๊กตามได้
```

ถ้าข้อมูลไม่ตรง:

```text
Dashboard แสดงตัวเลขไม่ตรงกับไฟล์ CSV
ในไฟล์มี [จำนวน] แถว
ในระบบเห็น [จำนวน] รายการ

ช่วยไล่สาเหตุที่เป็นไปได้จากง่ายไปยาก
เช่น filter, รายการซ้ำ, field ตัวเลข, RLS, query
```

## 10. ใช้ AI ตอนตั้งค่า AI Chatbot

```text
ฉันกำลังทดสอบ AI Chatbot ของระบบ
ต้องการให้ตอบจากข้อมูลจริง ไม่เดา และไม่เปิดเผยข้อมูลส่วนบุคคล

ช่วยทำชุดคำถามทดสอบ 10 ข้อ:
- คำถามทั่วไป
- คำถามจากข้อมูลใน dashboard
- คำถามที่ไม่ควรตอบเพราะเสี่ยงข้อมูลส่วนบุคคล
- คำถามที่ควรตอบว่าไม่มีข้อมูล
```

ใช้ตรวจคำตอบ:

```text
นี่คือคำถามผู้ใช้:
[วางคำถาม]

นี่คือคำตอบ chatbot:
[วางคำตอบ โดยลบข้อมูลส่วนบุคคลออก]

ช่วยตรวจว่า:
1. ตอบเกินข้อมูลที่มีไหม
2. มีข้อมูลส่วนบุคคลหลุดไหม
3. ควรปรับคำตอบให้ปลอดภัยขึ้นอย่างไร
```

```text
ฉันจะ deploy เว็บขึ้น Netlify เป็นครั้งแรก
ฉันไม่เคยใช้ Netlify

4. publish directory คืออะไร
5. environment variables ต้องใส่อะไร
6. หลัง deploy ต้องตรวจ URL ไหน

ใช้ค่าตัวอย่างปลอมเท่านั้น ห้ามขอ key จริง
```

ถ้า build fail:

```text
Netlify build fail
นี่คือ log ส่วน error:

ช่วยแปล error และบอกว่าต้องแก้ที่:
- package/dependency
- build command
- env

- route redirect
อย่างไหนน่าจะเป็นสาเหตุ
```

## 12. ใช้ AI ตอนดูแลระบบประจำวัน

```text
ฉันเป็นผู้ดูแลระบบประจำจังหวัด
วันนี้ต้องเช็กระบบแบบเร็ว

ช่วยทำ checklist รายวัน 10 นาที:
- เว็บเปิดได้
- login ได้
- dashboard โหลดได้
```

สรุป incident:

```text
นี่คือข้อมูล incident:

ช่วยสรุปเป็น:

- สาเหตุที่เป็นไปได้
```

## 14. Prompt ตรวจความปลอดภัยก่อนเผยแพร่

ใช้ก่อนเปิดหน้า public หรือส่งไฟล์ให้คนอื่น

```text
ฉันกำลังจะเผยแพร่ข้อมูลนี้:
[อธิบายประเภทข้อมูล ไม่ต้องวางข้อมูลจริง]

ช่วยตรวจความเสี่ยง:
- มีข้อมูลส่วนบุคคลไหม
- ควรตัด field ไหนออก
- ควรทำเป็น public/internal/restricted
- ต้องขออนุมัติก่อนเผยแพร่ไหม
- ถ้าจะให้ AI chatbot ใช้ข้อมูลนี้ ต้องระวังอะไร
```

## 15. ถาม AI อย่างไรให้ได้คำตอบที่ทำตามได้

ใช้หลักนี้ทุกครั้ง:

- บอกบทบาท: "ฉันเป็นเจ้าหน้าที่มือใหม่"
- บอกเป้าหมาย: "ต้องการเปิดเว็บ local"
- บอกสิ่งที่เห็น: วาง error หรืออธิบายหน้าจอ
- ขอคำตอบทีละขั้น
- ขอสิ่งที่ควรเห็นหลังทำถูก
- ขอจุดตรวจถ้าทำไม่สำเร็จ
- บอกข้อห้าม: ห้ามลบข้อมูล ห้ามใช้ key จริง ห้ามเดา

ตัวอย่างสั้น:

```text
ฉันเป็นเจ้าหน้าที่มือใหม่ กำลังทำขั้นตอน pnpm install
ตอนนี้ขึ้น error ตามนี้:
[error]

ช่วยแปล error เป็นภาษาไทยง่าย ๆ
แล้วบอกวิธีแก้ทีละขั้น
ห้ามให้คำสั่งลบไฟล์หรือ reset git
```

## Checklist ใช้ AI อย่างปลอดภัย

- [ ] ลบ key/token/password ก่อนส่ง
- [ ] ลบข้อมูลส่วนบุคคลก่อนส่ง
- [ ] ถามให้ AI ตอบทีละขั้น
- [ ] ให้ AI บอกผลที่ควรเห็น
