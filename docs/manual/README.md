# คู่มือการสร้างระบบศูนย์ข้อมูลการเกษตรจังหวัดด้วย NPT Dashboard

คู่มือนี้อธิบายกระบวนการสร้างระบบ `npt_dashboard` ตั้งแต่การเตรียมเครื่อง ดาวน์โหลดโค้ด เปิดเว็บในเครื่อง ตั้งค่า Supabase เตรียมข้อมูล ทำ Dashboard ตั้งค่า Search และ AI Chatbot ไปจนถึงการ Deploy และการดูแลระบบหลังเปิดใช้งาน

เป้าหมายสำคัญคือให้เจ้าหน้าที่หรือนักวิชาการเกษตรที่ไม่ถนัดงานคอมพิวเตอร์มาก่อน สามารถอ่านและทำตามได้ทีละขั้น โดยไม่ต้องเข้าใจโค้ดทั้งหมดตั้งแต่วันแรก

## กลุ่มผู้อ่านหลัก

- เจ้าหน้าที่ส่งเสริมการเกษตรที่รับผิดชอบข้อมูลจังหวัด
- นักวิชาการเกษตรที่ต้องการศึกษาและทดลองทำระบบด้วยตนเอง
- เจ้าหน้าที่ข้อมูลระดับอำเภอหรือจังหวัดที่คุ้นกับ Excel แต่ไม่เคยทำเว็บ
- ผู้ดูแลระบบของสำนักงานเกษตรจังหวัด
- ทีมพัฒนา Dashboard หรือระบบข้อมูลภายในหน่วยงาน
- ผู้บริหารที่ต้องการเข้าใจภาพรวมของระบบ
- จังหวัดอื่นที่ต้องการนำแนวทางนี้ไปปรับใช้

## ถ้าเริ่มจากศูนย์ ให้อ่านตามลำดับนี้

ลำดับนี้เหมาะสำหรับผู้อ่านที่ยังไม่เคยติดตั้งเว็บ ไม่เคยใช้ Supabase หรือไม่เคย Deploy เว็บมาก่อน

1. `00-เริ่มต้นสำหรับมือใหม่-ทำเว็บตั้งแต่ศูนย์จนออนไลน์.md`
2. `12-troubleshooting-สำหรับมือใหม่.md`
3. `01-ภาพรวมและเป้าหมายระบบ.md`
4. `02-การรวบรวมข้อมูลจากจังหวัด.md`
5. `03-การทำความสะอาดและเตรียมข้อมูล.md`
6. `04-การออกแบบฐานข้อมูลและตั้งค่า-supabase.md`
7. `05-การติดตั้งและตั้งค่าโปรเจกต์.md`
8. `09-เทคนิคการสั่ง-ai-ช่วยทำงานทุกขั้นตอน.md`
9. `07-ความปลอดภัยและการ-deploy.md`
10. `08-การดูแลระบบและอบรมผู้ใช้งาน.md`

แนวคิดของคู่มือชุดมือใหม่คือ "ทำให้เว็บเปิดได้ก่อน แล้วค่อยเข้าใจรายละเอียดลึกขึ้น" เพราะผู้อ่านจำนวนมากจะเรียนรู้ได้ดีขึ้นเมื่อเห็นผลลัพธ์จริงบนหน้าจอ

## ภาพรวมกระบวนการทั้งหมด

1. เตรียมเครื่อง โปรแกรม และบัญชีที่ต้องใช้
2. ดาวน์โหลดโค้ดจาก GitHub
3. ติดตั้ง dependencies และเปิดเว็บในเครื่อง
4. สร้าง Supabase project
5. ตั้งค่า `.env` เพื่อเชื่อมเว็บกับ Supabase
6. สร้างตาราง นำเข้าข้อมูล และตั้งค่า user/role
7. ทดสอบ Dashboard, Search, Login และ AI Chatbot
8. ตรวจสอบข้อมูลที่เผยแพร่ได้และข้อมูลที่ต้องจำกัดสิทธิ์
9. Build ระบบ
10. Deploy ขึ้น Netlify
11. ตรวจเว็บจริงหลังออนไลน์
12. วางรอบอัปเดตข้อมูล อบรมผู้ใช้ และดูแลระบบต่อเนื่อง

## หลักการเขียนคู่มือชุดนี้

- เขียนให้คนที่ไม่ใช่นักพัฒนาอ่านตามได้
- เริ่มจากงานที่ต้องทำจริง ไม่เริ่มจากทฤษฎี
- อธิบายคำศัพท์เทคนิคทันทีเมื่อจำเป็นต้องใช้
- ทุกบทควรบอกว่า "ถ้าทำถูกจะเห็นอะไร"
- ทุกบทควรบอกว่า "ถ้าทำไม่สำเร็จให้ตรวจจุดไหนก่อน"
- แยกงานข้อมูล งานฐานข้อมูล งานโค้ด และงานดูแลระบบให้ชัด
- ทุกขั้นตอนควรมี checklist ตรวจสอบ
- ส่วนที่เกี่ยวกับข้อมูลราชการต้องมีคำเตือนเรื่องสิทธิ์และความปลอดภัย
- เมื่อเนื้อหาเริ่มยาว ให้แตกหัวข้อย่อยออกเป็นไฟล์ใหม่ในอนาคต
