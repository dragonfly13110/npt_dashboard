# คู่มือการสร้างระบบศูนย์ข้อมูลการเกษตรจังหวัดด้วย NPT Dashboard

คู่มือนี้เป็นแผนเอกสารสำหรับอธิบายกระบวนการสร้างระบบ `npt_dashboard` ตั้งแต่การเตรียมข้อมูล การทำความสะอาดข้อมูล การออกแบบฐานข้อมูล การตั้งค่า Supabase การติดตั้งโปรเจกต์ การทำ Dashboard การตั้งค่า Search และ AI Chatbot ไปจนถึงการ Deploy และการดูแลระบบหลังเปิดใช้งาน

## กลุ่มผู้อ่านหลัก

- เจ้าหน้าที่ส่งเสริมการเกษตรที่รับผิดชอบข้อมูลจังหวัด
- ผู้ดูแลระบบของสำนักงานเกษตรจังหวัด
- ทีมพัฒนา Dashboard หรือระบบข้อมูลภายในหน่วยงาน
- ผู้บริหารที่ต้องการเข้าใจภาพรวมของระบบ
- จังหวัดอื่นที่ต้องการนำแนวทางนี้ไปปรับใช้

## ภาพรวมกระบวนการทั้งหมด

1. กำหนดเป้าหมายและขอบเขตของระบบ
2. สำรวจชุดข้อมูลที่จังหวัดมีอยู่
3. ขอข้อมูลจากกลุ่มงานและหน่วยงานที่เกี่ยวข้อง
4. ทำความสะอาดข้อมูลและปรับรูปแบบให้พร้อมใช้
5. ออกแบบฐานข้อมูลและนำข้อมูลเข้า Supabase
6. ตั้งค่า Authentication, Role, Department และ RLS
7. ติดตั้งโปรเจกต์ React + Vite บนเครื่องพัฒนา
8. เชื่อมข้อมูลกับ Dashboard, Search และ AI Chatbot
9. ตรวจสอบความปลอดภัยก่อนเปิดใช้งานจริง
10. Deploy ขึ้น Netlify
11. อบรมผู้ใช้และวางรอบการอัปเดตข้อมูล

## หลักการเขียนคู่มือชุดนี้

- เขียนให้คนที่ไม่ใช่นักพัฒนาอ่านตามได้
- แยกงานข้อมูล งานฐานข้อมูล งานโค้ด และงานดูแลระบบให้ชัด
- ทุกขั้นตอนควรมี checklist ตรวจสอบ
- ส่วนที่เกี่ยวกับข้อมูลราชการต้องมีคำเตือนเรื่องสิทธิ์และความปลอดภัย
- เมื่อเนื้อหาเริ่มยาว ให้แตกหัวข้อย่อยออกเป็นไฟล์ใหม่ในอนาคต
