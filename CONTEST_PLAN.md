# แผนส่งประกวดโครงการเพิ่มประสิทธิภาพงานส่งเสริมการเกษตรด้วยเทคโนโลยีและนวัตกรรม

## ภาพรวมโครงการประกวด

โครงการนี้เป็นเวทีของกรมส่งเสริมการเกษตร เพื่อค้นหาและต่อยอดนวัตกรรมที่ช่วยเพิ่มประสิทธิภาพงานส่งเสริมการเกษตรจากการทำงานจริงในพื้นที่ โดยเน้นผลงานที่แก้ปัญหาได้จริง ใช้เทคโนโลยีอย่างเหมาะสม ลดขั้นตอน ลดเวลา ลดทรัพยากร และมีโอกาสขยายผลสู่หน่วยงานหรือพื้นที่อื่น

ผลงานที่ส่งควรแสดงให้ชัดว่า:

- ปัญหาเดิมคืออะไร และกระทบงานส่งเสริมการเกษตรอย่างไร
- นวัตกรรมแก้ปัญหานั้นอย่างไร
- มีผลลัพธ์หรือประโยชน์ที่วัดได้หรืออธิบายได้
- มีความใหม่ ความคิดสร้างสรรค์ และเหมาะกับบริบทพื้นที่
- ทีมงานมีส่วนร่วม และมีโอกาสต่อยอดหรือขยายผล

## แนวคิดผลงานของโปรเจกต์นี้

ชื่อผลงานที่เสนอใช้เป็นร่าง:

**NPT Smart Agri Dashboard: ระบบศูนย์ข้อมูลและผู้ช่วย AI เพื่อยกระดับงานส่งเสริมการเกษตรจังหวัดนครปฐม**

โปรเจกต์นี้คือระบบเว็บสำหรับรวมข้อมูลเกษตรของจังหวัดนครปฐมไว้ในจุดเดียว มี Dashboard แสดงภาพรวม, ระบบแผนที่/ข้อมูลรายพื้นที่, ผู้ช่วย AI สำหรับถามตอบข้อมูล และแนวทางระบบขอข้อมูลจากอำเภอแบบตาราง เพื่อลดงานรวบรวมข้อมูลซ้ำ ๆ จากไฟล์ Excel หรือเอกสารกระจัดกระจาย

แกนเรื่องที่ควรเล่าในการประกวด:

- จากงานข้อมูลกระจายหลายแหล่ง สู่ศูนย์ข้อมูลเกษตรจังหวัดแบบใช้งานจริง
- จากการค้นหา/รวบรวมข้อมูลด้วยมือ สู่ Dashboard และ AI assistant
- จากการขอข้อมูลผ่านไฟล์หลายรอบ สู่ระบบ Data Request ที่จัดโครงสร้างข้อมูลได้
- จากการตัดสินใจด้วยข้อมูลล่าช้า สู่การดูภาพรวมและตอบคำถามได้เร็วขึ้น

## Pain Point ที่ควรใช้

Pain point หลัก:

- ข้อมูลเกษตรอยู่หลายแหล่ง หลายรูปแบบ เช่น Excel, Google Sheet, รายงาน, ระบบกลาง, ข้อมูลภาคสนาม
- เจ้าหน้าที่ใช้เวลามากในการรวบรวม ตรวจสอบ และสรุปข้อมูล
- ผู้บริหารหรือเจ้าหน้าที่ต้องการข้อมูลเร็ว แต่ข้อมูลพร้อมใช้งานไม่ทันเวลา
- ข้อมูลรายอำเภอ/รายกลุ่มงานไม่เชื่อมกัน ทำให้เห็นภาพรวมจังหวัดยาก
- การตอบคำถามเกษตรกรหรือหน่วยงานต้องพึ่งคนที่จำข้อมูลได้ หรือค้นเอกสารเอง

ผลกระทบที่ควรเขียน:

- เสียเวลาในการทำรายงานซ้ำ
- ลดเวลาที่ควรใช้กับงานส่งเสริมในพื้นที่
- การตัดสินใจเชิงพื้นที่ล่าช้า
- ข้อมูลไม่เป็นมาตรฐาน ทำให้เปรียบเทียบหรือวิเคราะห์ต่อยาก

ตัวเลขที่ควรเก็บเพิ่ม:

- เวลาก่อนใช้ระบบ: ใช้กี่ชั่วโมง/กี่วันในการรวมข้อมูลหนึ่งชุด
- เวลาหลังใช้ระบบ: เหลือกี่นาที/กี่ชั่วโมง
- จำนวนไฟล์หรือแหล่งข้อมูลที่รวมเข้าระบบ
- จำนวนเจ้าหน้าที่/กลุ่มงาน/อำเภอที่ทดลองใช้
- จำนวน dashboard หรือชุดข้อมูลที่พร้อมใช้งาน

## วัตถุประสงค์

- พัฒนาระบบศูนย์กลางข้อมูลเกษตรจังหวัดนครปฐมสำหรับสนับสนุนงานส่งเสริมการเกษตร
- ลดระยะเวลาและขั้นตอนการรวบรวม วิเคราะห์ และนำเสนอข้อมูล
- เพิ่มความสะดวกในการเข้าถึงข้อมูลของเจ้าหน้าที่ ผู้บริหาร และผู้เกี่ยวข้อง
- ใช้ AI เป็นผู้ช่วยค้นหา สรุป และตอบคำถามจากข้อมูลเกษตร
- สร้างต้นแบบที่ขยายผลไปยังอำเภอหรือจังหวัดอื่นได้

## เป้าหมาย

พื้นที่ดำเนินการ:

- จังหวัดนครปฐม
- สำนักงานเกษตรจังหวัดนครปฐม
- กลุ่มงานและอำเภอที่เกี่ยวข้อง

กลุ่มเป้าหมาย:

- เจ้าหน้าที่ส่งเสริมการเกษตร
- ผู้บริหารระดับจังหวัด/กลุ่มงาน
- เจ้าหน้าที่อำเภอ
- เกษตรกรหรือผู้รับบริการที่ต้องการข้อมูลเกษตร

ผลลัพธ์เป้าหมาย:

- มีระบบ Dashboard ใช้งานได้จริง
- มี AI assistant ช่วยตอบคำถามจากข้อมูล
- มีระบบจัดการข้อมูลและสิทธิ์ผู้ใช้งาน
- มีแนวทาง Data Request สำหรับขอข้อมูลจากอำเภอ
- มีหลักฐานการใช้งาน เช่น screenshot, demo, คลิป, infographic

## จุดเด่นนวัตกรรม

- รวมข้อมูลหลายมิติไว้ในระบบเดียว เช่น ข้อมูลพื้นที่ ผลผลิต ราคา สภาพอากาศ ภัยพิบัติ หรือข้อมูลสนับสนุนอื่น
- ใช้ AI ช่วยให้เจ้าหน้าที่ถามข้อมูลด้วยภาษาธรรมชาติได้
- Dashboard ช่วยให้เห็นภาพรวมและเจาะรายละเอียดได้เร็ว
- รองรับการจัดการสิทธิ์ตามบทบาท เช่น admin, editor, viewer
- ต่อกับ Supabase เป็น backend ทำให้จัดเก็บข้อมูลและต่อยอดระบบได้
- แนวทาง Data Request ช่วยเปลี่ยนการขอข้อมูลแบบไฟล์กระจัดกระจายให้เป็นข้อมูลโครงสร้าง

## รายการที่ต้องเตรียมตามใบสมัคร

ข้อมูลทั่วไป:

- ชื่อผลงาน
- ชื่อทีม
- หน่วยงาน
- รายชื่อทีมงานไม่เกิน 7 ราย
- ที่ปรึกษา
- ผู้ประสานงานหลัก
- เบอร์โทร, email, Line ID

รายละเอียดผลงาน:

- Pain Point พร้อมข้อมูลสนับสนุน
- วัตถุประสงค์
- เป้าหมาย
- กระบวนการดำเนินงาน
- ผลสัมฤทธิ์และคุณค่าของนวัตกรรม
- จุดเด่น
- ผลลัพธ์/ผลกระทบ
- ประโยชน์ต่อภาคเกษตรและองค์กร
- การลดขั้นตอน เวลา งบประมาณ หรือทรัพยากร
- โอกาสขยายผล
- การทำงานเป็นทีมและการบูรณาการ

หลักฐานประกอบ:

- Infographic กรอบแนวคิดของผลงาน
- ภาพกิจกรรมหรือภาพการใช้งานระบบ
- คลิปสั้น 1 นาที แนะนำผลงาน
- Screenshot หน้าระบบจริง
- Demo script สำหรับนำเสนอ
- Link ระบบ demo ถ้ามี

## แผนทำงานเพื่อส่งประกวด

### Phase 1: กำหนดเรื่องเล่า

- สรุปชื่อผลงาน
- เลือก pain point หลัก 1-2 เรื่อง
- เขียน one-liner ของนวัตกรรม
- กำหนดกลุ่มเป้าหมายและผลลัพธ์ที่ต้องการโชว์

ผลลัพธ์ Phase 1:

- คำอธิบายผลงาน 1 ย่อหน้า
- โครงใบสมัคร
- รายการตัวเลขที่ต้องหาเพิ่ม

### Phase 2: ตรวจระบบและเลือกฟีเจอร์โชว์

ฟีเจอร์ที่ควรโชว์:

- Landing page / ภาพรวมระบบ
- Dashboard ข้อมูลเกษตรจังหวัด
- Chatbot ผู้ช่วย AI
- หน้าแผนที่หรือข้อมูลเชิงพื้นที่
- ระบบผู้ใช้และสิทธิ์
- Data Request จาก branch `codex-data-requests-excel` ถ้าพร้อมนำเข้า

สิ่งที่ต้องตรวจ:

- ระบบเปิด demo ได้
- หน้า dashboard ไม่พัง
- ข้อมูลตัวอย่างพร้อม
- AI ตอบคำถามพื้นฐานได้
- Supabase table/policy ไม่บล็อก flow สำคัญ
- ไม่มีข้อมูลลับใน frontend หรือเอกสารส่งประกวด

### Phase 3: ทำใบสมัคร

เขียนตาม template:

- ตอนที่ 1 ข้อมูลทั่วไป
- ตอนที่ 2 รายละเอียดผลงาน
- สรุปผลลัพธ์แบบอ่านง่าย
- ใส่ตัวเลขก่อน/หลังใช้ระบบ

ต้องระวัง:

- อย่าเล่าเป็นแค่ “ทำเว็บ”
- ต้องเล่าว่า “แก้ปัญหางานส่งเสริมการเกษตรอย่างไร”
- ต้องมีผลลัพธ์หรือประโยชน์เชิงงานราชการ

### Phase 4: ทำสื่อประกอบ

Infographic ควรมีโครง:

- ปัญหาเดิม
- วิธีแก้ด้วย NPT Smart Agri Dashboard
- ฟีเจอร์หลัก
- ผลลัพธ์
- โอกาสขยายผล

คลิป 1 นาทีควรมีโครง:

- 0-10 วินาที: pain point
- 10-25 วินาที: แนะนำระบบ
- 25-45 วินาที: demo dashboard + AI + data request
- 45-55 วินาที: ผลลัพธ์
- 55-60 วินาที: ชื่อทีม/หน่วยงาน/ประโยชน์

### Phase 5: เตรียม Pitch

Pitch 3-5 นาทีควรเล่า:

- ปัญหาเดิม
- ทำไมต้องทำตอนนี้
- ระบบทำอะไรได้
- Demo flow สั้น
- ผลลัพธ์ที่เกิดขึ้น
- แผนขยายผล

## ข้อความสรุปผลงานฉบับร่าง

NPT Smart Agri Dashboard เป็นระบบศูนย์กลางข้อมูลเกษตรจังหวัดนครปฐมที่พัฒนาขึ้นเพื่อแก้ปัญหาข้อมูลกระจัดกระจาย การรวบรวมข้อมูลล่าช้า และการเข้าถึงข้อมูลเพื่อการตัดสินใจที่ยังไม่สะดวก ระบบรวม Dashboard, ข้อมูลเชิงพื้นที่, ผู้ช่วย AI และแนวทางการขอข้อมูลจากอำเภอไว้ในแพลตฟอร์มเดียว ช่วยให้เจ้าหน้าที่และผู้บริหารเห็นภาพรวมสถานการณ์เกษตรได้รวดเร็ว ลดขั้นตอนการทำรายงาน และต่อยอดสู่ระบบสนับสนุนงานส่งเสริมการเกษตรที่ขยายผลได้ในอนาคต
