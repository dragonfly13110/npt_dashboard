# รายงานตรวจรับคลังองค์ความรู้ฉบับสมบูรณ์ รุ่น 3.5

## ผลการตรวจรับ

**ผ่านการตรวจรับเชิงโครงสร้างสำหรับใช้เป็นคลัง Markdown เว็บไซต์ ฐานค้นหา และระบบ RAG**

**ผ่านแบบมีเงื่อนไขสำหรับการให้คำแนะนำภาคสนาม** เนื่องจากต้องตรวจทะเบียน ฉลาก PHI ประกาศกฎหมาย และสถานการณ์ความต้านทานล่าสุดก่อนใช้จริง

## ขอบเขตที่ส่งมอบ

- เนื้อหาพื้นฐาน ความปลอดภัย สูตรผลิตภัณฑ์ และกลไกการออกฤทธิ์
- คำแนะนำรายพืชด้านโรค แมลง ไร สัตว์ศัตรูพืช และวัชพืช
- สถานการณ์ความต้านทานและโปรแกรมหมุนเวียนสาร
- การใช้สารร่วมกับชีวภัณฑ์และสารธรรมชาติ
- เทคนิคเครื่องพ่นและอากาศยานไร้คนขับจากงานวิจัย
- บัญชีวัตถุอันตรายห้ามใช้ตามต้นฉบับ
- ดรรชนี ตารางความเข้ากันได้ ผู้จัดทำและคณะผู้วิจัย
- เอกสารระบบสำหรับตรวจสอบ ความครอบคลุม และควบคุมความเสี่ยง

## แนวทางนำเข้า RAG

1. ใช้หนึ่งไฟล์เป็นหนึ่ง document และแบ่ง chunk ตามหัวข้อ Markdown
2. เก็บ YAML เป็น metadata โดยเฉพาะ plant, pest_type, status, source_pages และกลุ่มกลไก
3. ให้ระบบค้นกรองพืชและศัตรูพืชก่อนค้นแบบ semantic
4. เพิ่มน้ำหนักไฟล์รายพืชมากกว่าไฟล์ดรรชนีเมื่อใช้ตอบอัตราสาร
5. ไม่ใช้ไฟล์ใน `99_ระบบ` เป็นแหล่งอัตราใช้ แต่ใช้สร้างคำเตือนและควบคุมคุณภาพคำตอบ
6. เมื่อ status มีธงตรวจสอบ ให้แนบคำเตือนและไม่สร้างค่าทดแทนเอง
7. บังคับคำตอบให้ระบุสูตร ความเข้มข้น อัตรา หน่วย วิธีใช้ และหน้าต้นฉบับพร้อมกัน

## เงื่อนไขการบำรุงรักษา

- ตรวจทะเบียนและกฎหมายอย่างน้อยปีละครั้ง หรือเมื่อมีประกาศใหม่
- เก็บ version ของคลังและบันทึกวันที่แก้ทุกไฟล์
- เมื่อแก้ชื่อสารหรือหน่วยจากหลักฐานใหม่ ต้องบันทึกค่าต้นฉบับ ค่าที่แก้ แหล่งยืนยัน และวันที่แก้
- รันชุดตรวจ YAML, slug, links, MANIFEST และ ZIP ทุกครั้งก่อนเผยแพร่รุ่นใหม่
