# สถานะการจัดทำคลังองค์ความรู้

ปรับปรุงล่าสุด: 19 กรกฎาคม 2569

## สถานะโครงการ

การแปลงเอกสารต้นฉบับเป็นคลังองค์ความรู้เสร็จสมบูรณ์ และรุ่น 3.6 ได้จัดทำชุดส่งออกสำหรับระบบ RAG พร้อมโครงสร้างฐานข้อมูล คำสั่งระบบ และชุดทดสอบ

## ภาพรวมคลังต้นทาง

- ไฟล์เนื้อหาองค์ความรู้จากเอกสารต้นฉบับ: **178 ไฟล์**
- หัวข้อความรู้ย่อยที่เรียบเรียงแล้ว: **ประมาณ 392 หัวข้อ**
- ไฟล์ Markdown รวมทั้งระบบหลังเพิ่มคู่มือ RAG: **191 ไฟล์**

## ผลการส่งออก RAG รุ่น 3.6

- เอกสารที่นำไปแบ่งข้อมูล: **178 เอกสาร**
- ชิ้นข้อมูลสำหรับ embedding: **1409 chunks**
- ชุดคำถามประเมิน: **70 กรณี**
- รูปแบบข้อมูล: JSONL, CSV, JSON และ SQL
- ค่าเป้าหมายขนาด chunk: ประมาณ 1,800 อักขระ
- ตาราง Markdown ถูกเก็บเป็นบล็อกเดียวเพื่อลดความเสี่ยงจับคู่สารและอัตราผิด

## เงื่อนไขก่อนนำไปใช้จริง

- เปลี่ยนมิติเวกเตอร์ใน SQL ให้ตรง embedding model
- ทดสอบ retrieval และ generation แยกกัน
- ห้ามยืนยันทะเบียน กฎหมาย หรือ PHI ปัจจุบันจากคลังคงที่
- ห้ามย้ายอัตราข้ามสูตรหรือความเข้มข้น
- เก็บ trace ของ chunks ที่ใช้ตอบทุกครั้ง

## งานบำรุงรักษา

- รันชุดทดสอบ regression เมื่อแก้ข้อมูลหรือเปลี่ยนโมเดล
- สร้าง embedding ใหม่เฉพาะเอกสารที่ค่า SHA-256 เปลี่ยน
- ทบทวน `REVIEW_FLAGS.md` และสถานะกฎหมายตามรอบ
