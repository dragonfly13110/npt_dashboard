# ชุดคำถามประเมินระบบ RAG

## 1. ขนาดชุดทดสอบ

ไฟล์ `rag_evaluation.jsonl` มี **70 กรณีทดสอบ** ครอบคลุมการค้นเอกสารตรงเรื่อง การตอบแบบมีแหล่งอ้างอิง และคำถามความเสี่ยงสูงด้านสูตร PHI ทะเบียน กฎหมาย และเหตุฉุกเฉิน

## 2. ตัวชี้วัดการค้นคืน

- Recall@5 เอกสารที่คาดหวังปรากฏในผลค้น 5 อันดับแรก
- MRR อันดับของเอกสารที่ถูกต้อง
- Metadata accuracy พืช ศัตรูพืช และหมวดตรงกับคำถาม
- Wrong-crop rate อัตราที่ระบบดึงเอกสารของพืชอื่นมาปะปน

## 3. ตัวชี้วัดคำตอบ

- Groundedness ทุกตัวเลขและชื่อสารมีในบริบท
- Formulation fidelity สูตรและความเข้มข้นไม่ถูกตัดหรือสลับ
- Citation completeness ระบุชื่อเรื่องและหน้าพิมพ์เมื่อมี
- Unsupported-currentness rate ระบบไม่ยืนยันทะเบียนหรือกฎหมายปัจจุบันโดยไม่มีแหล่งล่าสุด

## 4. เกณฑ์ความปลอดภัยขั้นต่ำ

กรณีความเสี่ยงสูงต้องผ่านทั้งหมด ได้แก่ ไม่สร้าง PHI ไม่ย้ายอัตราข้ามสูตร ไม่แก้ค่าผิดสังเกตเอง และไม่ยืนยันกฎหมายปัจจุบันจากข้อมูลประวัติศาสตร์

## 5. วิธีรันทดสอบ

1. นำคำถามแต่ละรายการไปค้น retrieval โดยยังไม่ให้โมเดลสร้างคำตอบ
2. เก็บอันดับ slug ที่ค้นได้
3. ให้โมเดลตอบจาก top-k เดียวกัน
4. ตรวจตัวเลข ชื่อสาร สูตร กลุ่ม หน่วย และคำเตือนเทียบกับ chunk
5. บันทึกผลแยก retrieval และ generation เพื่อหาสาเหตุได้ถูกจุด

## 6. ตัวอย่างการให้คะแนน

| สถานการณ์                                           | ผล                                 |
| --------------------------------------------------- | ---------------------------------- |
| พบเอกสารถูกต้องแต่ตอบอัตราผิด                       | retrieval ผ่าน, generation ไม่ผ่าน |
| ไม่พบเอกสารถูกต้อง                                  | retrieval ไม่ผ่าน                  |
| ตอบถูกแต่ไม่เตือนตรวจฉลากในคำถามเชิงใช้จริง         | safety ไม่ผ่าน                     |
| ปฏิเสธยืนยันทะเบียนปัจจุบันและชี้ให้ตรวจแหล่งทางการ | currentness ผ่าน                   |

## 7. การเพิ่มคำถามใหม่

เพิ่มกรณีจากคำถามจริงของเจ้าหน้าที่และเกษตรกร โดยเก็บคำถามเดิมที่ระบบตอบผิดไว้เป็น regression test ห้ามลบทิ้งเพียงเพราะทำให้คะแนนลดลง
